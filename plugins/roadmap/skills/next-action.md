# Next Action - 다음 액션 추천

현재 위치와 목표를 기반으로 지금 바로 해야 할 작업을 선별하고 구체적인 실행 가이드를 제공합니다.

## 트리거

`/next` 명령에서 `progress-locate.md` 완료 후 호출됩니다.

## 입력

- `outputs/roadmap/goal.md` (필수)
- `outputs/roadmap/roadmap.md` (필수)
- `outputs/roadmap/progress.md` (필수 - 직전 스킬에서 생성됨)
- `outputs/roadmap/alignment.md` (있으면 참고)
- 코드베이스 현재 상태

## 실행 내용

### 1. 진행 리포트 분석

`progress.md`에서 미완료 Step 목록과 현재 위치를 읽습니다.

### 2. 다음 작업 선별

미완료 Step 중 가장 적합한 1~2개를 선별합니다. 선별 기준 (우선순위 순):

1. **선행 조건 충족**: 이전 Step이 모두 완료된 것
2. **목표 기여도**: 핵심 목표에 직접 기여하는 것 우선
3. **ROADMAP 순서**: 순서상 다음인 것
4. **이전 정렬 이슈 반영**: `alignment.md`에서 지적된 문제가 있으면 반영

### 3. 작업 구체화

선별된 각 작업을 실행 가능한 수준으로 분해합니다:

- ROADMAP의 Step 설명을 읽고, 실제로 무엇을 해야 하는지 세부 작업으로 분해
- 코드베이스를 분석하여 어떤 파일을 수정/생성해야 하는지 파악
- 완료 기준을 명확히 정의
- 예상 산출물 명시

### 4. 하지 말아야 할 것 식별

현재 단계에서 흔히 저지르는 실수나, 범위를 벗어나는 작업을 경고합니다:

- 아직 하면 안 되는 최적화나 리팩토링
- 현재 Phase와 무관한 기능 추가
- `goal.md`의 Non-goals에 해당하는 작업

### 5. 리포트 작성

`outputs/roadmap/next-action.md`에 작성합니다.

## 산출물

- `outputs/roadmap/next-action.md`

## 완료 조건

- 다음 할 일이 1~2개 구체적으로 제시됨
- 각 작업의 세부 실행 항목이 나열됨
- 각 작업이 핵심 목표와 연결됨
- 완료 기준이 명확함
- "하지 말아야 할 것"이 포함됨

## 다음 스킬

없음 (명령 종료) — 사용자에게 추천된 작업 실행을 안내
