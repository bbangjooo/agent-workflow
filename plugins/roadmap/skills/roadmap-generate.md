# Roadmap Generate - ROADMAP 생성

정의된 목표를 기반으로 단계별 ROADMAP을 생성합니다.

## 트리거

`goal-capture.md` 완료 후 호출됩니다.

## 입력

- `outputs/roadmap/goal.md` (필수)
- 코드베이스 현재 상태 (있는 경우)
- 기존 `outputs/roadmap/roadmap.md` (업데이트 시)

## 실행 내용

### 1. 목표 분석

`outputs/roadmap/goal.md`를 읽고 핵심 목표들을 추출합니다.

### 2. 코드베이스 상태 분석 (있는 경우)

코드베이스가 이미 존재하면 현재 상태를 분석하여 이미 달성된 부분을 파악합니다:

- 프로젝트 구조, 주요 파일, 구현된 기능
- Git 히스토리를 통한 진행 경과 파악
- 기존 산출물 확인

### 3. Phase 설계

각 핵심 목표를 달성하기 위한 Phase를 설계합니다. 설계 원칙:

- **각 Phase는 독립적으로 가치를 제공해야 합니다** - Phase 완료 시 동작하는 결과물이 있어야 함
- **Phase 간 의존성은 순방향이어야 합니다** - Phase N은 Phase N-1의 결과물 위에 구축
- **모든 Phase는 최소 하나의 핵심 목표에 기여해야 합니다**
- **Phase는 3~7개, 각 Phase 내 Step은 3~7개가 적정합니다**

### 4. Step 분해

각 Phase를 구체적인 Step으로 분해합니다. 각 Step은:

- 하나의 명확한 작업을 설명
- 완료 여부를 객관적으로 판단할 수 있어야 함
- 1~3일 내에 완료 가능한 크기

### 5. 정렬 자체 검증

생성된 ROADMAP이 목표를 빠짐없이 커버하는지 스스로 검증합니다:

- 모든 핵심 목표에 기여하는 Phase/Step이 존재하는가?
- 불필요한 작업이 포함되어 있지 않은가?
- 성공 기준을 달성할 수 있는가?

### 6. ROADMAP 작성

`outputs/roadmap/roadmap.md`에 작성합니다.

### 7. 사용자 확인

작성된 ROADMAP을 사용자에게 보여주고 확인을 받습니다. 수정이 필요하면 반영합니다.

## 산출물

- `outputs/roadmap/roadmap.md`

## 완료 조건

- 모든 Phase가 최소 하나의 핵심 목표와 연결됨
- 각 Step이 구체적이고 완료 판단이 가능함
- 핵심 목표를 빠짐없이 커버함
- 사용자가 ROADMAP을 승인함
