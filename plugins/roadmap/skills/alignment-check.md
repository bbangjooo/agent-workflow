# Alignment Check - 목표 정렬 검증

ROADMAP의 모든 항목이 최초 목표에 실제로 기여하는지 검증하고, 방향 이탈을 감지합니다.

## 트리거

`/align` 명령 실행 시 호출됩니다.

## 입력

- `outputs/roadmap/goal.md` (필수)
- `outputs/roadmap/roadmap.md` (필수)
- `outputs/roadmap/progress.md` (있으면 참고)
- 코드베이스 현재 상태

## 실행 내용

### 1. 사전 조건 확인

`outputs/roadmap/goal.md`와 `outputs/roadmap/roadmap.md`가 모두 존재하는지 확인합니다. 없으면 `/roadmap`을 먼저 실행하라고 안내하고 종료합니다.

### 2. 목표와 ROADMAP 로드

두 파일을 모두 읽고 핵심 목표와 모든 Phase/Step을 파싱합니다.

### 3. 순방향 검증 (ROADMAP → 목표)

각 Step에 대해 질문합니다: **"이 작업은 어떤 핵심 목표에 기여하는가?"**

- 직접 기여: 핵심 목표를 직접적으로 달성하는 데 필요한 작업
- 간접 기여: 다른 Step을 가능하게 하여 궁극적으로 목표에 기여
- 무관: 어떤 목표에도 기여하지 않음

무관한 작업은 🔴로 표시합니다.

### 4. 역방향 검증 (목표 → ROADMAP)

각 핵심 목표에 대해 질문합니다: **"이 목표를 달성하기 위한 작업이 ROADMAP에 충분히 있는가?"**

- 충분: 목표 달성에 필요한 모든 작업이 있음
- 부족: 일부 작업이 누락됨
- 없음: 이 목표에 기여하는 작업이 전혀 없음

누락된 작업은 🟢으로 표시합니다.

### 5. 우선순위 검증

현재 ROADMAP의 실행 순서가 적절한지 검증합니다:

- 의존성 순서가 맞는가?
- 핵심 목표에 가장 크게 기여하는 작업이 앞에 있는가?
- 리스크가 높은 작업이 뒤로 밀려있지 않은가?

문제가 있으면 🟡로 표시합니다.

### 6. 실제 작업 대비 검증 (progress.md가 있는 경우)

지금까지 실제로 한 작업이 ROADMAP 순서와 맞는지, 예상치 못한 작업이 추가되지 않았는지 확인합니다:

- ROADMAP에 없는데 진행한 작업이 있는가?
- ROADMAP 순서를 건너뛴 작업이 있는가?

### 7. 정렬 점수 산출

위 분석을 종합하여 A~F 점수를 부여합니다:

- **A**: 모든 작업이 목표에 직접 기여, 누락 없음
- **B**: 95% 이상 정렬, 경미한 조정만 필요
- **C**: 80% 이상 정렬, 일부 작업 제거/추가 필요
- **D**: 60% 이상 정렬, 상당한 수정 필요
- **F**: 60% 미만 정렬, ROADMAP 재수립 필요

### 8. 방향 수정 제안

문제가 발견된 경우, 구체적인 수정 액션을 제안합니다:

- 제거할 작업과 이유
- 추가할 작업과 기여 목표
- 순서 변경이 필요한 작업

### 9. 리포트 작성

`outputs/roadmap/alignment.md`에 작성합니다.

## 산출물

- `outputs/roadmap/alignment.md`

## 완료 조건

- 모든 핵심 목표에 대해 기여도 분석이 완료됨
- 모든 Step에 대해 목표 연결이 검증됨
- 정렬 점수가 부여됨
- 문제 발견 시 구체적인 수정 제안이 포함됨
