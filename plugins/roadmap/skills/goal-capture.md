# Goal Capture - 프로젝트 목표 파악

프로젝트의 핵심 목표를 사용자와의 대화를 통해 파악하고 명확하게 정의합니다.

## 트리거

`/roadmap` 명령 실행 시 첫 번째로 호출됩니다.

## 입력

- 사용자와의 대화
- 기존 코드베이스 (있는 경우)
- 기존 `outputs/roadmap/goal.md` (있는 경우)

## 실행 내용

### 1. 기존 목표 확인

`outputs/roadmap/goal.md`가 이미 존재하는지 확인합니다.

- **존재하는 경우**: 내용을 읽고 사용자에게 보여준 후, 수정이 필요한지 묻습니다.
- **존재하지 않는 경우**: 목표 파악 대화를 시작합니다.

### 2. 목표 파악 대화

사용자에게 다음을 순서대로 질문합니다. 이미 알고 있는 정보는 건너뜁니다.

1. **"이 프로젝트로 뭘 만들려고 하세요?"** - 핵심 목표 파악
2. **"이걸 왜 만드세요? 어떤 문제를 해결하나요?"** - 배경과 맥락 파악
3. **"이게 성공했다고 볼 수 있는 기준이 뭔가요?"** - 성공 기준 파악
4. **"명시적으로 하지 않을 것이 있나요?"** - 범위 밖 정의
5. **"시간, 기술, 리소스 등 제약이 있나요?"** - 제약 조건 파악

### 3. 코드베이스 분석 (보조)

코드베이스가 이미 존재하는 경우, 다음을 분석하여 목표 파악을 보조합니다:

- `README.md`, `package.json`의 description 등
- 프로젝트 구조와 주요 기능
- 기존 `outputs/` 디렉토리의 산출물

### 4. 목표 정의서 작성

수집한 정보를 바탕으로 `outputs/roadmap/goal.md`를 작성합니다.

### 5. 사용자 확인

작성된 목표 정의서를 사용자에게 보여주고 확인을 받습니다. 수정이 필요하면 반영합니다.

## 산출물

- `outputs/roadmap/goal.md`

## 완료 조건

- 핵심 목표가 구체적이고 측정 가능한 형태로 정의됨
- 성공 기준이 명시됨
- 범위 밖(Non-goals)이 정의됨
- 사용자가 목표 정의서를 승인함
