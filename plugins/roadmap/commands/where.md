# /where

ROADMAP 상의 현재 위치를 파악하고, 지금까지 한 일과 앞으로 할 일을 정리합니다.

## 사용법

```
/where    # 현재 위치 파악
```

## 설명

코드베이스, 커밋 히스토리, 기존 산출물 등을 분석하여 ROADMAP 상에서 현재 어디까지 진행되었는지 파악합니다. 완료된 작업과 남은 작업을 구분하고, 바로 다음에 해야 할 일을 구체적으로 제시합니다.

## 트리거 키워드

- "지금 어디야"
- "어디까지 했어"
- "뭐 해야 해"
- "다음에 뭐 해"
- "진행 상황 알려줘"
- "현재 상태"

## 실행 스킬 순서

1. `progress-locate.md` - 현재 진행 상황을 분석하고 `outputs/roadmap/progress.md`에 저장

## Agent

`roadmap-coach.md`

## 입력

- `outputs/roadmap/roadmap.md` (필수 - 없으면 `/roadmap`부터 실행하라고 안내)
- `outputs/roadmap/goal.md` (필수)
- 코드베이스 현재 상태
- Git 커밋 히스토리
- 기존 `outputs/` 디렉토리의 다른 산출물

## 산출물

- `outputs/roadmap/progress.md` - 진행 상황 리포트

### progress.md 형식

```markdown
# 진행 상황 리포트

> 생성 일시: [날짜]

## 현재 위치
**Phase [N]: [이름]** - Step [N.M]

## 전체 진행률
[====>          ] Phase 1 ████████░░ 80%
[               ] Phase 2 ░░░░░░░░░░ 0%
...

## 완료된 작업
### Phase 1: [이름]
- [x] Step 1.1: [작업] — [근거: 어떤 코드/커밋/산출물로 확인]
- [x] Step 1.2: [작업] — [근거]
- [ ] Step 1.3: [작업] — 미완료

## 다음 할 일 (바로 다음 1~3개)
1. **[가장 우선순위 높은 작업]** - [왜 이걸 먼저 해야 하는지]
2. [다음 작업]
3. [그 다음 작업]

## 남은 작업 요약
- Phase [N] 잔여: [M]개 Step
- Phase [N+1] 전체: [K]개 Step
- ...

---
마지막 업데이트: [날짜]
```

## 완료 조건

- `outputs/roadmap/progress.md`가 생성됨
- 각 Step의 완료/미완료 판단에 근거(코드, 커밋, 산출물)가 포함됨
- 바로 다음에 할 작업이 구체적으로 제시됨
