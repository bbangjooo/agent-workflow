# /align

ROADMAP의 각 항목이 최초 목표에 실제로 기여하는지 검증하고, 방향 이탈을 감지합니다.

## 사용법

```
/align    # 전체 목표 정렬 검증
```

## 설명

ROADMAP의 모든 Phase와 Step이 `goal.md`에 정의된 핵심 목표에 실제로 기여하는지 하나하나 검증합니다. 목표와 무관한 작업, 우선순위가 잘못된 작업, 빠진 작업을 식별하고, 방향 수정을 제안합니다.

## 트리거 키워드

- "방향 맞아?"
- "목표랑 맞는지 확인해줘"
- "이거 하는 게 맞아?"
- "삼천포로 빠진 거 아냐"
- "정렬 검증"
- "얼라인 체크"

## 실행 스킬 순서

1. `alignment-check.md` - 목표 정렬을 검증하고 `outputs/roadmap/alignment.md`에 저장

## Agent

`roadmap-coach.md`

## 입력

- `outputs/roadmap/goal.md` (필수)
- `outputs/roadmap/roadmap.md` (필수)
- `outputs/roadmap/progress.md` (있으면 참고)
- 코드베이스 현재 상태

## 산출물

- `outputs/roadmap/alignment.md` - 목표 정렬 검증 리포트

### alignment.md 형식

```markdown
# 목표 정렬 검증 리포트

> 생성 일시: [날짜]

## 정렬 점수: [A/B/C/D/F]

- **A**: 모든 작업이 목표에 직접 기여
- **B**: 대부분 정렬됨, 경미한 이탈
- **C**: 일부 작업이 목표와 무관
- **D**: 상당한 방향 이탈 감지
- **F**: ROADMAP 재수립 필요

## 핵심 목표별 기여도

### 목표 1: [목표 내용]
- 기여하는 작업: [Step 목록]
- 기여도: [높음/중간/낮음]
- 누락된 것: [이 목표를 달성하려면 빠진 작업]

### 목표 2: [목표 내용]
...

## 문제 발견

### 🔴 목표와 무관한 작업
- [Step X.Y]: [작업 내용] — [왜 목표와 무관한지 설명]

### 🟡 우선순위 재조정 필요
- [Step X.Y]: [작업 내용] — [왜 순서가 잘못되었는지]

### 🟢 누락된 작업
- [추가해야 할 작업] — [어떤 목표에 기여하는지]

## 방향 수정 제안

1. [구체적인 수정 액션 1]
2. [구체적인 수정 액션 2]
3. ...

## 결론

[전반적인 판단과 권고사항을 2~3문장으로]

---
마지막 업데이트: [날짜]
```

## 완료 조건

- `outputs/roadmap/alignment.md`가 생성됨
- 모든 핵심 목표에 대해 기여도 분석이 완료됨
- 문제가 발견된 경우 구체적인 수정 제안이 포함됨
- 정렬 점수가 부여됨

## 다음 단계

- 점수가 C 이하인 경우: `/roadmap update` - ROADMAP 수정
- 점수가 B 이상인 경우: `/where` - 다음 작업 확인 후 진행
- 누락된 작업이 있는 경우: ROADMAP에 추가
