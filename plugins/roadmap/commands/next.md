# /next

목표와 ROADMAP을 파악하고, 현재 위치를 분석한 뒤, 지금 바로 해야 할 일을 추천합니다.

## 사용법

```
/next    # 다음 할 일 추천
```

## 설명

하나의 명령으로 전체 흐름을 실행합니다: 목표 확인 → 현재 위치 분석 → 목표 정렬 확인 → 다음 액션 추천. 이미 `goal.md`와 `roadmap.md`가 있으면 재활용하고, 없으면 생성부터 시작합니다. 최종적으로 "지금 바로 해야 할 일"을 구체적으로 제시합니다.

## 트리거 키워드

- "다음 뭐 해"
- "지금 뭐 하면 돼"
- "다음 할 일 알려줘"
- "뭐부터 하면 돼"
- "오늘 뭐 하지"

## 실행 스킬 순서

1. `goal-capture.md` - 목표 확인 (기존 `goal.md`가 있으면 읽기만 함)
2. `roadmap-generate.md` - ROADMAP 확인 (기존 `roadmap.md`가 있으면 읽기만 함)
3. `progress-locate.md` - 현재 위치 분석
4. `next-action.md` - 다음 액션을 선별하고 구체적인 실행 가이드 제공

## Agent

`roadmap-coach.md`

## 입력

- `outputs/roadmap/goal.md` (있으면 재활용)
- `outputs/roadmap/roadmap.md` (있으면 재활용)
- 코드베이스 현재 상태
- Git 커밋 히스토리

## 산출물

- `outputs/roadmap/goal.md` (없었으면 생성)
- `outputs/roadmap/roadmap.md` (없었으면 생성)
- `outputs/roadmap/progress.md` (갱신)
- `outputs/roadmap/next-action.md` (생성)

### next-action.md 형식

```markdown
# 다음 액션

> 생성 일시: [날짜]

## 현재 위치 요약
**Phase [N]: [이름]** — [한 줄 요약]

## 지금 해야 할 일

### 1. [가장 우선순위 높은 작업]
- **ROADMAP 위치**: Phase [N] > Step [N.M]
- **기여 목표**: [어떤 핵심 목표에 기여하는지]
- **구체적으로 할 것**:
  - [실행 가능한 수준의 세부 작업 1]
  - [세부 작업 2]
  - [세부 작업 3]
- **완료 기준**: [이것이 되면 이 작업은 끝]
- **예상 산출물**: [파일, 기능, 결과물 등]

### 2. [그 다음 작업] (선택)
...

## 하지 말아야 할 것
- [지금 단계에서 하면 안 되는 것과 이유]

## 완료 후
이 작업을 마치면 `/where`로 진행 상황을 갱신하세요.

---
마지막 업데이트: [날짜]
```

## 완료 조건

- 목표와 ROADMAP이 존재함 (기존 것 또는 새로 생성)
- 현재 위치가 분석됨
- 다음 할 일이 실행 가능한 수준으로 구체화됨
- 각 추천 작업이 핵심 목표와 연결됨
