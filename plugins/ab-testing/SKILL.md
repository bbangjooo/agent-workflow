# AB Test Manager (AB 테스트 매니저)

AB 테스트의 전체 라이프사이클을 관리하는 플러그인입니다. **실험 주제를 간단히 말하면, 가설 설계부터 결과 분석 프레임워크까지 전체 실험을 구상합니다.**

## 개요

사용자가 "회원가입 버튼 색상을 바꿔보고 싶어"처럼 간단한 주제만 전달하면, AB Test Coach가 체계적인 실험 전체를 설계합니다. 여러 프로젝트에서 각각 다수의 실험을 독립적으로 관리할 수 있습니다.

> 핵심: 감에 의한 의사결정이 아니라, 데이터 기반 의사결정을 위한 실험 설계. "이것도 해보고 저것도 해보자"가 아니라 "이 가설을 이 지표로 검증한다"는 명확한 실험.

## 사용 방법

```
/ab-test                          — 새 실험 설계 시작
/ab-test list                     — 진행 중/완료된 실험 목록
/ab-test analyze {experiment-id}  — 기존 실험 결과 분석
```

## 주요 컴포넌트

### Command

- **/ab-test**: AB 테스트 설계 시작 및 관리

### Agent

- **AB Test Coach**: 실험 설계부터 분석까지 안내하는 코치 에이전트

### Skills

| 스킬 | 설명 |
|------|------|
| hypothesis-design | 현상 분석, 가설 수립, 변형 정의, 성공 지표(KPI) 설정 |
| experiment-plan | 샘플 사이즈 계산, 기간 산정, 트래픽 분배, 기술 구현 가이드 |
| analysis-framework | 통계적 유의성 판단 기준, 결과 해석 가이드, 의사결정 프레임워크 |
| experiment-log | 실험 로그 기록, 학습점 축적, 후속 실험 추천 |

## 산출물 구조

프로젝트별, 실험별로 독립 관리됩니다:

```
outputs/ab-tests/
  {project-name}/
    experiments.md              ← 프로젝트 실험 목록 (인덱스)
    {experiment-id}/
      hypothesis.md             ← 가설 설계
      experiment-plan.md        ← 실험 실행 계획
      analysis-framework.md     ← 결과 분석 프레임워크
      experiment-summary.md     ← 최종 실험 요약 및 학습점
```

## 실험 ID 규칙

- 형식: `exp-{번호}-{짧은설명}` (예: `exp-001-signup-button-color`)
- 프로젝트 내에서 순번 자동 증가
