# Hypothesis Design

Step 1: 가설 설계

## 설명

모호한 실험 아이디어를 구체적이고 검증 가능한 가설로 전환하는 스킬입니다. 변형(Variant)을 정의하고, 성공을 측정할 KPI를 설정합니다.

## 트리거

- `/ab-test` 실행 시 Step 1으로 자동 호출

## 입력

- 사용자의 실험 주제 (자연어)
- 프로젝트 컨텍스트 (있는 경우)
- 기존 실험 히스토리 (있는 경우)

## 실행 내용

### 1. 현상 분석 (As-is)

사용자와 대화하며 현재 상태를 파악합니다:

- "현재 {대상}은 어떤 상태인가요?"
- "이걸 바꾸고 싶은 이유가 있나요? (데이터, 유저 피드백, 직감 등)"
- "현재 {관련 지표}는 어느 정도인가요?" (알고 있다면)

### 2. 가설 수립

구조화된 가설문을 작성합니다:

```
가설 형식:
"만약 [독립변수: 무엇을 변경]하면,
 [종속변수: 어떤 지표가]
 [방향: 증가/감소]할 것이다.
 왜냐하면 [근거: 변경이 효과를 낼 이유]이기 때문이다."

예시:
"만약 회원가입 폼을 3단계에서 1단계로 줄이면,
 가입 완료율이 20% 이상 증가할 것이다.
 왜냐하면 현재 2단계에서 이탈률이 45%로 높기 때문이다."
```

### 3. 변형(Variant) 정의

```
- Control (A): 현재 상태 — 변경 없음
  → 상세 명세: {현재 UI/로직/카피 등 구체적 설명}

- Treatment (B): 변경 버전
  → 상세 명세: {변경할 UI/로직/카피 등 구체적 설명}
  → 변경 포인트: {정확히 무엇이 달라지는지}

- Treatment (C): (선택적, 필요 시만)
  → 상세 명세: {또 다른 변형}
```

**변형 설계 원칙:**
- 한 번에 하나의 변수만 변경 (다변량은 명시적으로 선택할 때만)
- Control과 Treatment의 차이가 명확하고 구체적이어야 함
- 변형이 너무 많으면 필요 트래픽이 급증 — 2-3개 권장

### 4. 성공 지표(KPI) 설정

```
Primary KPI (주요 지표 — 반드시 1개):
- 지표명: {예: 가입 완료율}
- 현재 값: {예: 12%} (알고 있다면)
- 목표 방향: {증가/감소}
- 최소 감지 효과(MDE): {예: 상대적 20% 개선 → 12% → 14.4%}

Secondary KPI (보조 지표 — 1-3개):
- {예: 가입 시작률, 가입 소요 시간}
- 주의: Secondary는 의사결정에 참고만, Primary로 판단

Guardrail Metrics (가드레일 — 1-2개):
- {예: 페이지 이탈률, 고객 문의 건수}
- 이 지표가 X% 이상 악화되면 실험 중단
```

## 산출물

`outputs/ab-tests/{project}/{exp-id}/hypothesis.md`

```markdown
# Hypothesis: {실험 제목}

## 메타데이터
- 실험 ID: {exp-id}
- 프로젝트: {project-name}
- 생성일: {날짜}
- 상태: designing

## 현상 분석 (As-is)

### 현재 상태
{현재 UI/프로세스/로직 설명}

### 변경 근거
{왜 변경이 필요한지 — 데이터, 피드백, 관찰 등}

### 관련 데이터 (있는 경우)
{현재 전환율, 이탈률, 클릭률 등}

## 가설

> **만약** {독립변수}하면,
> **{종속변수}가** {방향}할 것이다.
> **왜냐하면** {근거}이기 때문이다.

## 변형 (Variants)

### Control (A) — 현재 버전
{상세 명세}

### Treatment (B) — 변경 버전
{상세 명세}

**변경 포인트:**
- {구체적 차이 1}
- {구체적 차이 2}

## 성공 지표

### Primary KPI
| 지표 | 현재 값 | 목표 방향 | MDE |
|------|---------|----------|-----|
| {지표명} | {값} | {↑/↓} | {%} |

### Secondary KPI
| 지표 | 역할 |
|------|------|
| {지표 1} | {왜 보조 지표로 설정했는지} |

### Guardrail Metrics
| 지표 | 악화 기준 | 대응 |
|------|----------|------|
| {지표} | {X% 이상 악화 시} | 실험 중단 |

## 이전 실험과의 관계
{기존 실험에서 배운 것이 이 실험에 어떻게 반영되었는지. 첫 실험이면 "없음"}
```

## 완료 조건

- 구조화된 가설문 작성 완료
- Control + Treatment 최소 1개 변형 정의
- Primary KPI 1개 확정
- Guardrail 지표 최소 1개 설정
- hypothesis.md 파일 생성
