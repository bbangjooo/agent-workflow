# Experiment Log

Step 4: 실험 관리

## 설명

실험 전체를 한 눈에 볼 수 있는 요약을 생성하고, 프로젝트 실험 인덱스에 등록하는 스킬입니다. 학습점을 축적하고 후속 실험을 추천합니다.

## 트리거

- Step 3 (analysis-framework) 완료 후 자동 진행
- `/ab-test analyze {experiment-id}` 실행 시 (결과 기록용)

## 입력

- `hypothesis.md` — 가설 설계
- `experiment-plan.md` — 실험 계획
- `analysis-framework.md` — 분석 프레임워크
- `experiments.md` — 기존 프로젝트 실험 인덱스 (있는 경우)

## 실행 내용

### 1. 실험 요약 생성

Step 1~3의 핵심 내용을 한 페이지로 요약합니다:

```
실험 한 눈에 보기:
- 가설: {한 줄 요약}
- 변형: Control vs Treatment (핵심 차이 한 줄)
- Primary KPI: {지표}
- 기간: {N}주 (필요 샘플: {N}명)
- 분석: {방법론} (α={값})
- 승리 조건: {한 줄 요약}
```

### 2. 프로젝트 실험 인덱스 업데이트

`outputs/ab-tests/{project}/experiments.md`에 실험을 등록합니다:

```
신규 프로젝트:
- experiments.md 파일 새로 생성
- 프로젝트 헤더 및 실험 목록 초기화

기존 프로젝트:
- experiments.md 읽기
- 새 실험 항목 추가
- 실험 번호 순서 확인
```

### 3. 학습점 섹션 준비

```
실험 설계 시 (designed 상태):
- "이 실험에서 배우고 싶은 것" 섹션 작성
- 성공/실패 시 각각 무엇을 배울 수 있는지 정리

실험 완료 시 (completed 상태):
- 실제 결과와 예측 비교
- 예상과 다른 점은 무엇이었는지
- 다음 실험에 반영할 학습점
```

### 4. 후속 실험 추천

```
이 실험의 결과에 따라 자연스럽게 이어질 수 있는 실험 1-2개를 추천합니다:

추천 기준:
- Treatment 승리 시: 효과를 더 키울 수 있는 후속 최적화
- Control 승리 시: 다른 접근법으로 같은 문제 해결
- 무승부 시: 더 큰 변경 또는 다른 영역 테스트
- 가설에서 파생된 새로운 질문

형식:
- 후속 실험 1: "{제목}" — {한 줄 설명} (우선도: 높/중/낮)
- 후속 실험 2: "{제목}" — {한 줄 설명} (우선도: 높/중/낮)
```

### 5. 이전 실험과의 연결

```
같은 프로젝트의 이전 실험과 관계를 기록합니다:

- 동일 영역 이전 실험이 있는가?
- 이전 실험의 학습점이 이번 설계에 반영되었는가?
- 실험 간 모순되는 결과는 없는가?
```
