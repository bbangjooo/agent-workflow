# Experiment Plan

Step 2: 실험 실행 계획

## 설명

가설을 검증하기 위한 구체적인 실험 실행 계획을 수립하는 스킬입니다. 통계적으로 유의미한 결과를 얻기 위해 필요한 샘플 사이즈, 기간, 트래픽 분배를 계산합니다.

## 트리거

- Step 1 (hypothesis-design) 완료 후 자동 진행

## 입력

- `hypothesis.md` — Step 1에서 작성한 가설 설계
- 사용자 답변 (트래픽, 기술 환경 등)

## 실행 내용

### 1. 트래픽 파악

사용자에게 질문합니다:

- "실험 대상 페이지/기능의 **일일 방문자(또는 이벤트) 수**는 얼마나 되나요?"
- "모르겠다면 대략적인 범위라도 괜찮아요 (100 이하 / 100-1,000 / 1,000-10,000 / 10,000+)"

### 2. 샘플 사이즈 계산

```
입력값:
- Baseline conversion rate: {hypothesis.md의 현재 값}
- MDE (Minimum Detectable Effect): {hypothesis.md에서 설정}
- Significance level (α): 0.05 (기본값)
- Statistical power (1-β): 0.80 (기본값)
- 변형 수: {Control + Treatment 수}

계산 공식 (이항 비율 기준):
n = (Z_α/2 + Z_β)² × [p₁(1-p₁) + p₂(1-p₂)] / (p₁ - p₂)²

결과:
- 변형당 필요 샘플: {n}명
- 총 필요 샘플: {n × 변형 수}명
```

**트래픽 부족 시 대안 제시:**
- MDE를 키움 (더 큰 차이만 감지) → 기간 단축
- 유의수준을 0.10으로 완화 → 기간 단축 (트레이드오프 설명)
- Sequential testing 적용 → 조기 종료 가능
- AB 테스트 대신 Pre/Post 분석 또는 정성적 검증 제안

### 3. 실험 기간 산정

```
예상 기간 = 총 필요 샘플 ÷ 일일 트래픽

보정 요인:
- 요일 효과: 최소 1주(7일) 단위로 반올림
- 계절성: 특수 시기(연말, 명절 등) 포함 여부 확인
- 안정화 기간: 시작 후 1-2일은 안정화로 제외 고려

최종 권장 기간: {N}주 ({시작일} ~ {종료일} 권장)
```

### 4. 트래픽 분배 전략

```
기본 권장: 50% / 50% (Control / Treatment)
- 검정력 최대화에 유리

비균등 분배가 필요한 경우:
- 리스크 큰 변경 → 90/10 또는 80/20으로 시작 후 증가
- 변형이 3개 → 34/33/33
- 기존 유저 영향 최소화 → Control 비율을 높게

트래픽 분배 방식:
- 유저 단위 (권장): 같은 유저가 항상 같은 변형을 봄
- 세션 단위: 같은 유저도 세션마다 다른 변형 가능 (비권장)
- 페이지뷰 단위: 매 방문마다 변경 (비권장)
```

### 5. 세그먼트 고려사항

```
실험 대상:
- 포함: {전체 유저 / 특정 세그먼트}
- 제외: {있다면 — 예: 내부 직원, 봇 트래픽}

세그먼트별 분석 계획:
- 신규 vs 기존 유저
- 모바일 vs 데스크톱
- {제품 특성에 따른 세그먼트}

주의: 사전에 정의한 세그먼트만 분석 (사후 세그먼트 마이닝 금지)
```

### 6. 기술 구현 가이드

```
구현 방식:
- [ ] 서버 사이드 분기 (추천: 깜빡임 없음)
- [ ] 클라이언트 사이드 분기 (간편하지만 깜빡임 가능)
- [ ] Feature flag 서비스 사용 (LaunchDarkly, Statsig 등)
- [ ] 자체 구현

분기 지점:
- 파일/컴포넌트: {변경이 필요한 코드 위치}
- 분기 로직: {유저 ID 해싱 등}

이벤트 트래킹:
- 노출 이벤트: {실험 변형을 본 순간 기록}
- 전환 이벤트: {Primary KPI에 해당하는 이벤트}
- 가드레일 이벤트: {모니터링할 이벤트}
```

### 7. 시작/종료 조건

```
시작 조건:
- [ ] 코드 구현 및 QA 완료
- [ ] 이벤트 트래킹 검증 (A/B 모두 노출 이벤트 발화 확인)
- [ ] 팀 내 실험 계획 공유

종료 조건 (아래 중 하나):
- [ ] 목표 샘플 사이즈 도달
- [ ] 최대 실험 기간 경과 ({N}주)
- [ ] 가드레일 지표 위반 (즉시 중단)

조기 종료 규칙:
- 가드레일 위반 시 즉시 중단, Control로 롤백
- 중간 확인은 사전 정의된 시점에만 (peeking 방지)
```
