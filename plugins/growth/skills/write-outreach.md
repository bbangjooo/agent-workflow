# Write Outreach

타겟 인물(인플루언서, 기자, 커뮤니티 리더, 얼리어답터)에게 보낼 아웃리치 메시지를 생성합니다.

## 설명

제품과 관련된 **핵심 인물에게 직접 연락하는 메시지**(이메일, LinkedIn DM, Twitter/X DM, 커뮤니티 DM)를 작성합니다. 스팸이 아닌 **개인화된 메시지**로, 상대방의 관심사와 맥락을 반영하여 답장률을 높입니다.

> "100명에게 같은 메시지를 보내면 스팸이다. 10명에게 각자 맞춤 메시지를 보내면 네트워킹이다."

## 입력

- `outputs/promotion/channel-research.md` — 채널별 인물 정보
- `outputs/promotion/audience-language.md` — 유저 언어 톤
- `outputs/stage-5/market-research.md` — 경쟁사, 커뮤니티 리더
- `outputs/stage-0/idea-brief.md` — 제품 설명, 가치 제안
- `outputs/stage-1/prd.md` — 핵심 기능

## 실행 내용

### 1. 타겟 인물 리서치 (웹 서치)

```
제품과 관련된 핵심 인물을 식별하고 개인 정보를 수집합니다.

[인물 유형별 조사]

[인플루언서/크리에이터]
- 웹 서치로 "{카테고리} influencer/youtuber/blogger" 검색
- 웹 서치로 "{카테고리} twitter/x thought leader" 검색
- 웹 서치로 "{카테고리} 유튜버/블로거" 검색 (한국)
- 수집: 이름, 플랫폼, 팔로워 수, 최근 콘텐츠 주제, 연락 방법

[기자/에디터]
- 웹 서치로 "{카테고리} journalist/editor TechCrunch/TheVerge" 검색
- 웹 서치로 "{카테고리} 기자 벤처스퀘어/플래텀/아웃스탠딩" 검색 (한국)
- 수집: 이름, 매체, 최근 기사 제목, Beat(담당 분야), 연락처

[커뮤니티 리더/모더레이터]
- 타겟 서브레딧의 모더레이터 확인
- 관련 Slack/Discord 커뮤니티의 관리자 확인
- 네이버 카페 운영자 확인 (한국)
- 수집: 이름/닉네임, 커뮤니티, 역할, 활동 패턴

[잠재적 얼리어답터/파워유저]
- 경쟁사 Product Hunt 댓글에서 활발한 유저 식별
- 관련 Reddit/Twitter 토론에서 활발한 유저 식별
- "{문제 키워드}" 관련 블로그 글 작성자 식별
- 수집: 이름/닉네임, 플랫폼, 관심사, 연락 방법

[업계 전문가/어드바이저 후보]
- 웹 서치로 "{카테고리} advisor/expert/consultant" 검색
- LinkedIn에서 관련 분야 전문가 검색
- 수집: 이름, 직함, 회사, 전문 분야, LinkedIn URL

[각 인물별 수집 항목 — 연락처 필수]
1. 이름 (실명 또는 핸들)
2. 플랫폼 (이메일/LinkedIn/Twitter/Reddit 등)
3. **연락처 (필수)** — 아래 중 최소 1개를 반드시 찾아서 기록:
   - 이메일 주소 (공개된 것: 사이트 contact, GitHub 프로필, 블로그 about 등)
   - LinkedIn 프로필 URL
   - Twitter/X 프로필 URL
   - 기타 DM 가능한 플랫폼 핸들
   - 연락처를 못 찾으면 해당 인물 목록에서 제외하거나 "연락처 미확인" 명시
4. 최근 활동 (무엇에 관심 있는지)
5. 연결 고리 (우리 제품과 왜 관련 있는지)
6. 선호 연락 방법 (공개된 것 기준)

> **연락처 없는 인물은 산출물에 포함하지 않습니다.** 
> 메시지를 작성해도 보낼 곳이 없으면 의미가 없습니다.
```

### 2. 아웃리치 메시지 유형별 템플릿

```
[이메일 — 콜드 아웃리치]

제목: {개인화된 제목 — 상대방의 최근 콘텐츠/관심사 언급}
  좋은 예: "Your article on {주제} inspired us to build {제품}"
  나쁜 예: "Check out our new product!" (스팸)

본문 구조 (150단어 이내):
1. 개인화 인사 (상대방의 최근 활동 구체적 언급) — 1문장
2. 자기소개 + 제품 한줄 설명 — 1문장
3. 왜 연락하는지 (구체적 요청) — 1-2문장
4. 상대방에게 주는 가치 — 1문장
5. 부담 없는 CTA — 1문장

톤 규칙:
- 짧고 직접적 (장황한 소개 금지)
- 상대방 이름 사용
- "I" 보다 "you" 중심
- 구체적 요청 (리뷰, 피드백, 공유, 인터뷰 중 1가지만)
- 팔로업 시점 명시 안 함 (압박 금지)
```

```
[LinkedIn DM]

구조 (100단어 이내):
1. 연결 요청 메시지: 공통점 또는 상대방 콘텐츠 언급 — 2문장
2. (승인 후) 첫 메시지: 감사 + 제품 소개 + 구체적 요청 — 3-4문장

규칙:
- 연결 요청에 제품 링크 넣지 않음 (승인률 급감)
- 승인 후 최소 1일 대기 후 메시지
- 직접적 판매 금지 — 피드백/의견 요청 형태
```

```
[Twitter/X DM]

구조 (280자 이내):
1. 상대방 트윗/스레드 구체적 언급 — 1문장
2. 제품 소개 — 1문장
3. 요청 — 1문장

규칙:
- DM 전에 먼저 상대방 트윗에 의미 있는 댓글/인용 RT 2-3회
- 갑자기 DM 보내지 않음 (관계 먼저)
- 링크는 요청 시에만 공유
```

```
[Discord/Slack DM]

구조 (100단어 이내):
1. 자기소개 + 커뮤니티 활동 언급 — 1문장
2. 제품 소개 — 1문장
3. 질문/피드백 요청 — 1문장

규칙:
- DM 전에 해당 커뮤니티에서 최소 1주일 활동
- 관리자에게는 채널 홍보 허가 요청부터
- 일반 멤버에게는 피드백 요청 형태
```

```
[한국어 이메일/DM]

구조:
1. 인사 + 상대방 콘텐츠 언급 — 1문장
2. 자기소개 (간단히) — 1문장
3. 제품 설명 + 왜 연락드리는지 — 2문장
4. 구체적 요청 — 1문장
5. 감사 인사 — 1문장

톤 규칙:
- 존댓말 기본
- "바쁘신 와중에 연락드려 죄송합니다" 류의 과도한 사과 금지
- 간결하고 정중하게
- "혹시 관심 있으시면" 식의 부담 없는 CTA
```

### 3. 어프로치 프레이밍 (응답률 최적화)

```
메시지를 작성하기 전에, 각 인물에 대해 가장 응답률이 높은 접근법을 설계합니다.

[Step 3-1] 인물별 어프로치 리서치 (웹 서치)
- 웹 서치로 "{인물 이름} interview/podcast/AMA" 검색 → 성격/커뮤니케이션 스타일 파악
- 웹 서치로 "how to pitch to {매체/인물 유형}" 검색 → 업계 모범 사례
- 해당 인물이 과거에 공유/추천한 제품이 있으면 확인 → 그 제품과의 유사점 활용
- 상대방이 최근에 올린 글/트윗/영상에서 관심사 키워드 추출

[Step 3-2] 프레이밍 전략 결정
각 인물에 대해 아래 5가지 프레이밍 중 가장 효과적인 것을 선택합니다:

프레이밍 A: 피드백 요청 (Feedback Ask)
- "당신의 의견을 듣고 싶습니다"
- 적합: 업계 전문가, 얼리어답터
- 응답률: 높음 (부담 없고 전문성 인정)

프레이밍 B: 스토리 공유 (Story Share)
- "이런 문제를 겪었고 이렇게 풀었습니다"
- 적합: 기자, 콘텐츠 크리에이터 (기사/콘텐츠 소재 제공)
- 응답률: 중-높 (가치 제공)

프레이밍 C: 상호 이익 (Mutual Benefit)
- "당신의 오디언스에게도 도움이 될 것 같습니다"
- 적합: 인플루언서, 뉴스레터 운영자
- 응답률: 중 (명확한 가치 교환)

프레이밍 D: 감사/팬 (Appreciation)
- "당신의 {콘텐츠}에서 영감을 받아 만들었습니다"
- 적합: 크리에이터, 오픈소스 메인테이너
- 응답률: 높음 (진정성 있으면)

프레이밍 E: 독점/얼리액세스 (Exclusive Access)
- "공개 전에 먼저 써보실 수 있습니다"
- 적합: 리뷰어, 테크 블로거, 파워유저
- 응답률: 중-높 (독점 심리)

[Step 3-3] 채널 선택 (응답률 우선순위)
인물별로 가장 응답률이 높은 채널을 선택합니다:

| 인물 유형 | 1순위 채널 | 2순위 채널 | 근거 |
|----------|----------|----------|------|
| 기자 | 이메일 | Twitter DM | 기자는 이메일 피칭에 익숙 |
| 인플루언서 | Twitter DM | 이메일 | 이미 활동하는 플랫폼에서 접근 |
| 개발자 | Twitter DM/Email | GitHub issue | 기술 커뮤니티 문화 |
| 커뮤니티 리더 | 해당 커뮤니티 DM | 이메일 | 그 커뮤니티 안에서 접근 |
| 한국 블로거 | 이메일 | 블로그 댓글 | 정중한 이메일 선호 |

→ 웹 서치로 해당 인물의 공개 연락 선호도 확인 후 최종 결정
```

### 4. 인물별 개인화 메시지 작성

```
각 타겟 인물에 대해:

1. 웹 서치로 상대방의 최근 콘텐츠/활동 1개 확인
2. 우리 제품과의 연결 고리 파악
3. Step 3에서 결정한 프레이밍 & 채널 적용
4. 해당 채널 템플릿으로 개인화된 메시지 작성
5. 팔로업 메시지 1개 추가 작성 (첫 메시지 5-7일 후)
```

### 5. 아웃리치 에티켓 & 금지 행위

```
[필수 준수]
- 한 사람에게 최대 2회 연락 (첫 메시지 + 1회 팔로업)
- 팔로업은 최소 5일 후
- "No reply = No" — 2회 무응답이면 중단
- 개인화 필수 — 같은 메시지를 여러 명에게 보내지 않음
- BCC로 단체 이메일 금지

[금지 행위]
- 스팸성 대량 발송
- 연결 요청/DM에 바로 제품 링크 투척
- "혁신적인 제품을 소개합니다" 류의 과장 표현
- 상대방 답장 전에 팔로업 연발
- 유료 리뷰/포스팅 요청 (윤리 위반 명시하지 않는 한)
- 개인정보 무단 수집/사용
```

## 4. 아웃리치 실행 체크리스트

### 사전 준비 (D-Day 전)
- [ ] Twitter/X: 타겟 인물 팔로우 + 의미 있는 댓글 2-3회
- [ ] LinkedIn: 연결 요청 (제품 언급 없이)
- [ ] 커뮤니티: 타겟 커뮤니티에서 1주일 활동

### D-Day 아웃리치
- [ ] 1순위 인물에게 개인화 메시지 발송
- [ ] 기자에게 보도자료 + 피칭 이메일 발송

### D+5~7 팔로업
- [ ] 무응답 인물에게 팔로업 1회
- [ ] 응답한 인물에게 후속 대응
