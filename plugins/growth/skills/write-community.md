# Write Community Posts

IndieHackers, Dev.to, GeekNews, OKKY, Velog 등 커뮤니티 포스트를 생성합니다.

## 설명

각 커뮤니티의 톤과 포맷에 맞춘 포스트를 작성합니다. 국내/글로벌 커뮤니티 각각의 문화를 반영하여 바로 올릴 수 있는 형태로 출력합니다.

## 입력

> **참고**: 아래 리서치 산출물이 존재하면 반드시 참조하여 콘텐츠를 생성합니다:
> - `outputs/promotion/copy-research.md` — 경쟁사 카피 패턴 → 검증된 구조 활용
> - `outputs/promotion/channel-research.md` — 커뮤니티 규칙/인기글 패턴 → 규칙 준수 & 효과적 포맷
> - `outputs/promotion/audience-language.md` — 유저 실제 표현 → 유저 언어로 작성, 금지 용어 회피
> - `outputs/promotion/timing-research.md` — 최적 포스팅 시간 → 산출물에 추천 시간 명시

- 제품명, 한줄 설명, 핵심 기능, 빌딩 스토리, 기술 스택, 라이브 URL

## 실행 내용

### 1. IndieHackers

```
[포스트 유형] "I built..."

[제목]
"I built {제품명} — {한줄 설명} ({성과: 첫 N명 유저 / N일 만에 등})"

[본문 구조] (500-1000단어)
1. 배경: 왜 만들었는지 (개인 스토리)
2. 빌딩 과정: 기간, 기술 스택, 비용
3. 런칭: 어떻게 첫 유저를 얻었는지
4. 현재 상태: 유저 수, 매출 (공개 가능 시)
5. 배운 점: 핵심 교훈 3가지
6. 질문: 커뮤니티에 구체적 질문

톤: 투명하고 솔직, 숫자 공개, 실패도 공유
```

### 2. Dev.to

```
[포스트 유형] 기술 블로그 + 프로젝트 소개

[제목]
"I built {제품명} with {기술스택} — Here's what I learned"
또는 "How I built {기능} using {기술}"

[본문 구조] (800-1500단어)
1. 소개: 무엇을 만들었는지
2. 기술적 결정: 왜 이 스택을 선택했는지
3. 아키텍처: 시스템 구조 간략 설명
4. 챌린지: 기술적 어려움과 해결 과정
5. 코드 스니펫: 핵심 부분 코드 공유 (1-2개)
6. 결과: 제품 링크 + 데모
7. 오픈소스: 기여 환영 (해당 시)

태그: #webdev #javascript #react #{기술} #showdev
톤: 기술 교육적, 코드 포함
```

### 3. GeekNews (긱뉴스)

```
[포스트 유형] 링크 공유 + 한글 설명

[제목]
"{제품명} — {한줄 설명}" 또는 "{기술적 주제}를 해결하는 {제품명}"

[설명] (200-500자 한글)
- 무엇인지 1줄
- 기술적 특징 2-3줄
- 링크

톤: 간결하고 기술적, HN과 유사
```

### 4. OKKY

```
[포스트 유형] 사이드 프로젝트 공유

[제목]
"[사이드 프로젝트] {제품명} 만들었습니다 — {한줄 설명}"

[본문] (500-800자 한글)
1. 소개: 어떤 서비스인지
2. 동기: 왜 만들었는지
3. 기술 스택: 사용 기술 상세
4. 핵심 기능: 3-5개
5. 링크 + 피드백 요청

톤: 한국 개발자 커뮤니티, 기술 중심, 겸손한
```

### 5. Velog

```
[포스트 유형] 기술 블로그 (회고록 스타일)

[제목]
"{제품명} 개발기" 또는 "{기술}로 {제품} 만들기"

[본문] (1000-2000자 한글)
1. 배경과 동기
2. 기술 선택 과정
3. 개발 과정 (타임라인)
4. 기술적 챌린지와 해결
5. 배운 점
6. 서비스 소개 + 링크

태그: #{기술} #사이드프로젝트 #개발기 #{카테고리}
톤: 개발 회고록, 상세한 기술 내용
```

### 6. Slack/Discord 커뮤니티

```
[메시지 포맷] 짧고 직접적

"안녕하세요! {문제}를 해결하는 {제품명}을 만들었습니다.
{핵심 가치 1줄}
{URL}
피드백 환영합니다! 🙏"

톤: 간결, 스팸 느낌 금지, 채널 규칙 준수
```

- 각 커뮤니티 톤/포맷 준수
- 모든 포스트 바로 복붙 가능한 형태
