# Market Research

Step 6.1: 시장/경쟁사/커뮤니티 실제 조사

## 설명

프로젝트 컨텍스트를 기반으로 **웹 서치를 통해 실제 시장 데이터, 경쟁사, 타겟 유저 커뮤니티를 조사**하는 스킬입니다. 감이 아닌 데이터로 전략을 세울 수 있게 합니다.

> "페르소나를 상상으로 만들지 마라. 타겟 유저가 실제로 어디서 뭘 하고 있는지 찾아라."

## 트리거

- `/grow` 실행 시 프로젝트 컨텍스트 분석 직후 첫 번째로 실행
- 모든 이전 Stage 산출물 분석 완료 후

## 입력

- 프로젝트 컨텍스트 분석 결과 (growth-coach가 이전 산출물에서 추출)
- idea-brief.md → 제품 카테고리, 타겟 유저, 해결하는 문제
- prd.md → 핵심 기능, 경쟁사 언급 (있으면)

## 실행 내용

### 1. 경쟁사 실제 조사 (웹 서치)

```
[조사 방법]
- 웹 서치로 "{제품 카테고리} tool/app/service" 검색
- 웹 서치로 "{해결하는 문제} solution" 검색
- 웹 서치로 "best {카테고리} for {타겟 유저}" 검색
- 웹 서치로 "{경쟁사명} pricing/review" 검색 (알려진 경쟁사)

[조사 항목 — 경쟁사별]
1. 서비스명 + URL
2. 한줄 설명 (실제 사이트에서 추출)
3. 가격 모델 & 가격표 (실제 pricing 페이지 확인)
4. 주요 기능 리스트
5. 타겟 유저 (사이트에서 어떤 유저를 타겟하는지)
6. 포지셔닝 메시지 (헤드라인, 서브헤드라인)
7. 리뷰/평판 (G2, Capterra, ProductHunt, 앱스토어 리뷰)
8. 트래픽 추정 (가능 시: SimilarWeb, 관련 기사)
9. 사용자 수 추정 (공개된 수치, PR 기사)
10. 강점 / 약점 / 우리와의 차별점
```

### 2. 타겟 유저 커뮤니티 실제 탐색 (웹 서치)

```
[조사 방법]
- 웹 서치로 "{타겟 유저} community/forum" 검색
- 웹 서치로 "reddit {타겟 유저}/{문제 키워드}" 검색
- 웹 서치로 "{문제 키워드} site:reddit.com" 검색
- 웹 서치로 "{카테고리} 네이버 카페" 검색 (국내)
- 웹 서치로 "{카테고리} slack/discord community" 검색

[조사 항목 — 커뮤니티별]
1. 커뮤니티명 + URL
2. 플랫폼 (Reddit/네이버카페/Discord/Slack/Facebook그룹 등)
3. 멤버 수 / 활성도
4. 주요 토론 주제
5. 셀프프로모션 규칙 (홍보 가능 여부)
6. 유저들이 자주 불평하는 문제 (= 우리 제품의 기회)
7. 유저들이 추천하는 기존 도구 (= 경쟁사 확인)
```

### 3. 시장 규모 조사 (웹 서치)

```
[조사 방법]
- 웹 서치로 "{카테고리} market size 2025/2026" 검색
- 웹 서치로 "{카테고리} industry report" 검색
- 웹 서치로 "{카테고리} growth rate CAGR" 검색

[조사 항목]
1. 전체 시장 규모 (TAM) — 글로벌
2. 접근 가능한 시장 (SAM) — 타겟 세그먼트
3. 시장 성장률 (CAGR)
4. 주요 트렌드 / 시장 변화
5. 국내 시장 규모 (해당 시)
```

### 4. 유저 니즈/페인포인트 실제 조사 (웹 서치)

```
[조사 방법]
- 웹 서치로 "{문제 키워드} frustrated/hate/wish" 검색
- 웹 서치로 "{경쟁사} complaints/problems" 검색
- Reddit/HN에서 관련 토론 탐색

[조사 항목]
1. 유저들이 실제로 겪는 문제 (원문 인용)
2. 기존 도구에 대한 불만 (원문 인용)
3. 유저들이 원하지만 없는 기능
4. 지불 의향 관련 대화 (있으면)
```
