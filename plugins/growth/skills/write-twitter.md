# Write Twitter/X Launch

Twitter/X 런칭 스레드와 개별 트윗을 생성합니다.

## 설명

Tech Twitter 문화에 맞는 런칭 스레드, 빌드인퍼블릭 포스트, 개별 프로모 트윗을 생성합니다. 바로 복붙해서 올릴 수 있는 형태로 출력합니다.

## 입력

> **참고**: 아래 리서치 산출물이 존재하면 반드시 참조하여 콘텐츠를 생성합니다:
> - `outputs/promotion/copy-research.md` — 경쟁사 카피 패턴 → 검증된 구조 활용
> - `outputs/promotion/channel-research.md` — 커뮤니티 규칙/인기글 패턴 → 규칙 준수 & 효과적 포맷
> - `outputs/promotion/audience-language.md` — 유저 실제 표현 → 유저 언어로 작성, 금지 용어 회피
> - `outputs/promotion/timing-research.md` — 최적 포스팅 시간 → 산출물에 추천 시간 명시

- 제품명, 한줄 설명, 핵심 기능, 빌딩 스토리, 라이브 URL

## 실행 내용

### 1. 런칭 스레드 (7-10트윗)

```
트윗 1 (훅):
- 문제 또는 성과로 시작 (숫자가 있으면 좋음)
- "I just launched..." 또는 "After X months of building..."
- 280자 이내

트윗 2 (문제):
- 왜 만들었는지 (개인 경험)

트윗 3-5 (해결책 + 기능):
- 핵심 기능 각각 1트윗씩
- 스크린샷/GIF 첨부 권장

트윗 6 (빌딩 과정):
- 기술 스택, 소요 시간, 힘들었던 점

트윗 7 (배운 점):
- 빌딩 과정에서 배운 1가지

트윗 8 (가격/접근):
- 무료인지, 어떻게 시작하는지

트윗 9 (CTA):
- 구체적 요청: "써보고 피드백 주세요"
- 링크 포함

트윗 10 (감사):
- 영감 준 사람/도구 멘션
- RT/공유 요청
```

### 2. 개별 프로모 트윗 (5개 시안)

```
[기능 하이라이트]
"{기능}을 {N}초 만에 할 수 있다면?" + 스크린샷

[유저 문제 공감]
"{문제 상황}... 이거 나만 그런 거 아니죠?" + 해결책 제시

[비포/애프터]
"Before: {기존 방식}
After: {우리 제품 방식}" + 비교 이미지

[숫자/성과]
"런칭 {N}일 만에 {성과}" (있을 경우)

[빌드인퍼블릭]
"오늘 {기능}을 추가했습니다. {배운 점}" + 스크린샷
```

### 3. 해시태그 추천

```
#buildinpublic #indiehackers #SaaS #startup
+ 제품 도메인별 해시태그 3-5개
```
