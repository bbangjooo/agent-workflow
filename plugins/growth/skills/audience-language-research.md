# Audience Language Research

타겟 유저가 실제로 사용하는 단어, 표현, 검색어를 조사합니다.

## 설명

경쟁사가 쓰는 마케팅 용어가 아니라, **타겟 유저가 이 문제를 설명할 때 실제로 쓰는 말**을 웹 서치로 수집합니다. 커뮤니티 게시글, 리뷰, 검색어에서 유저의 실제 언어를 추출하여, write-* 스킬이 "광고 냄새" 없이 유저와 같은 말로 글을 쓸 수 있게 합니다.

> "마케터가 쓰는 말 ≠ 유저가 쓰는 말. '워크플로우 자동화 솔루션'이라고 쓰면 아무도 안 읽는다. 유저는 '매번 반복하는 귀찮은 작업'이라고 말한다."

## 트리거

- `/promote` 실행 시 `channel-research` 다음으로 실행
- `outputs/promotion/channel-research.md` 완료 후

## 입력

- `outputs/stage-5/market-research.md` — 경쟁사 목록, 커뮤니티 목록, 유저 페인포인트
- `outputs/stage-0/idea-brief.md` — 해결하는 문제, 타겟 유저
- `outputs/stage-1/prd.md` — 핵심 기능, 가치 제안

## 실행 내용

### 1. 커뮤니티 언어 수집 (웹 서치)

```
타겟 유저가 활동하는 커뮤니티에서 실제 표현을 수집합니다.

[조사 방법]
- 웹 서치로 "{문제 키워드} site:reddit.com" 검색 → 유저 게시글/댓글 인용
- 웹 서치로 "{문제 키워드} frustrated/annoying/wish" 검색
- 웹 서치로 "{문제 키워드} 불편/짜증/힘들" 검색 (한국어)
- 웹 서치로 "{카테고리} 추천 site:reddit.com" 검색
- 웹 서치로 "{카테고리} 추천" 네이버 카페/블로그 검색 (한국어)
- IndieHackers, HN, Twitter에서 관련 토론 탐색

[수집 항목 — 원문 인용 필수]
1. 문제를 설명하는 표현
   - "I'm so tired of..." / "매번 ...하는 게 너무 귀찮아"
   - "I wish there was a..." / "...하는 도구 없나요?"
   - "Does anyone else struggle with..." / "다들 어떻게 하세요?"

2. 기존 도구에 대한 불만 표현
   - "{경쟁사} is too..." / "{경쟁사}가 너무 ..."
   - "I switched from {경쟁사} because..." 
   - "{경쟁사} alternative" / "{경쟁사} 대안"

3. 원하는 해결책을 설명하는 표현
   - "All I need is..." / "그냥 ...만 되면 되는데"
   - "Something simple that..." / "간단하게 ...할 수 있는 거"
   - "Like {서비스} but for..." / "{서비스} 같은 건데 ...용"

4. 커뮤니티 특유의 은어/약어
   - 기술 커뮤니티: DX, devex, dogfooding, yak-shaving 등
   - 스타트업: PMF, PLG, ARR, churn, MoM 등
   - 한국: 갓생, 현타, 뇌피셜 등 (타겟에 따라)
```

### 2. 리뷰/평점 언어 수집 (웹 서치)

```
경쟁사 리뷰에서 유저의 실제 평가 언어를 수집합니다.

[조사 방법]
- 웹 서치로 "{경쟁사} review" 검색 → G2, Capterra, Trustpilot
- 웹 서치로 "{경쟁사} 후기/리뷰" 검색 (한국어)
- 앱스토어 리뷰 (모바일 앱일 경우)
- Product Hunt 댓글

[수집 항목 — 원문 인용 필수]
1. 긍정 리뷰에서 반복되는 표현
   - "Finally something that..." / "드디어 ...하는 게 나왔다"
   - "Love how it..." / "...하는 게 진짜 좋다"
   → 우리 제품에도 해당되면 그대로 활용 가능

2. 부정 리뷰에서 반복되는 불만
   - "Too complicated..." / "너무 복잡하다"
   - "Missing..." / "...기능이 없다"
   → 우리 제품의 차별점으로 활용

3. 별점 패턴
   - 5점 리뷰 공통 키워드
   - 1-2점 리뷰 공통 불만
```

### 3. 검색어 언어 수집 (웹 서치)

```
유저가 이 문제/해결책을 찾을 때 실제로 입력하는 검색어를 조사합니다.

[조사 방법]
- 웹 서치로 "{카테고리} 자동완성 검색어" 확인
- 웹 서치로 "people also ask {문제 키워드}" 검색
- 웹 서치로 "{문제 키워드}" 검색 후 관련 검색어 수집
- 웹 서치로 "{경쟁사} vs" 검색 → 유저가 비교하는 대상 파악
- 네이버 자동완성 + 연관검색어 확인 (한국어)

[수집 항목]
1. 문제 관련 검색어
   - "how to {문제}" / "{문제} 해결 방법"
   - "best tool for {문제}" / "{문제} 도구 추천"

2. 카테고리 검색어
   - "{카테고리} for {타겟}" / "{타겟}용 {카테고리}"
   - "free {카테고리}" / "무료 {카테고리}"
   - "open source {카테고리}"

3. 비교 검색어
   - "{경쟁사A} vs {경쟁사B}" 
   - "{경쟁사} alternative" / "{경쟁사} 대안"
   - "best {경쟁사} alternatives 2025/2026"

4. 의도별 분류
   - 정보 탐색: "what is {카테고리}" / "{카테고리}란"
   - 비교 검토: "{A} vs {B}" 
   - 구매 의도: "best {카테고리}" / "{카테고리} pricing"
```

### 4. 언어 패턴 종합

```
수집된 모든 표현을 분석하여 write-* 스킬이 활용할 수 있는 형태로 정리합니다.

[정리 항목]
1. 핵심 감정 키워드 (유저가 느끼는 감정)
   - frustration: "{원문}" 
   - hope: "{원문}"
   - relief: "{원문}"

2. 파워 프레이즈 (높은 공감을 얻는 표현)
   - "{높은 upvote/좋아요를 받은 표현}"
   → 그대로 차용하거나 변형

3. 금지 용어 (유저가 거부감을 느끼는 표현)
   - 마케팅 용어: "혁신적인", "game-changing", "revolutionary"
   - 과장 표현: "유일한", "최고의", "세계 최초"
   → 이런 표현은 write-* 스킬에서 사용 금지

4. 채널별 톤 매핑
   | 채널 | 유저 언어 톤 | 예시 |
   |------|-------------|------|
   | Reddit | 캐주얼, 직설적 | "honestly, {경쟁사} sucks at..." |
   | HN | 기술적, 비판적 | "interesting approach, but how does it handle..." |
   | Twitter | 짧고 강렬 | "just shipped X. no more Y." |
   | 네이버 | 친근, 후기 형식 | "써본 결과 솔직 후기..." |
```

## 완료 조건

- 커뮤니티 유저 표현 최소 10개 원문 인용 수집
- 경쟁사 리뷰 불만 표현 최소 5개 수집
- 검색어 클러스터 3가지(문제/비교/구매) 정리
- 파워 프레이즈 최소 5개, 금지 용어 최소 5개 정리
- 채널별 톤 가이드 작성
- `audience-language.md` 파일 생성

## 사용처

- 모든 `write-*` 스킬이 이 파일을 참조하여 유저의 언어로 글 작성
- 파워 프레이즈를 헤드라인/제목에 활용
- 금지 용어를 피하여 "광고 냄새" 제거
- 채널별 톤 가이드로 플랫폼 맞춤 글쓰기
