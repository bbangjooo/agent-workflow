# Channel Research

타겟 채널의 현재 트렌드, 인기 콘텐츠, 커뮤니티 규칙을 실제 조사합니다.

## 설명

글을 쓰기 전에 **해당 채널에서 지금 뭐가 통하고 있는지** 웹 서치로 실제 조사합니다. 각 채널의 최신 인기 콘텐츠 패턴, 커뮤니티 규칙, 자기홍보 정책을 파악하여 write-* 스킬이 규칙 위반 없이 효과적인 글을 작성할 수 있게 합니다.

> "채널의 규칙을 모르고 올리면 삭제당하거나 밴당한다. 먼저 규칙을 읽고, 인기글을 분석하고, 그다음에 글을 쓴다."

## 트리거

- `/promote` 실행 시 `copy-research` 다음으로 실행
- `outputs/promotion/copy-research.md` 완료 후

## 입력

- `outputs/stage-5/market-research.md` — 타겟 커뮤니티 목록
- `outputs/stage-5/market-analysis.md` — 타겟 시장 (국내/글로벌)
- `/promote` 에서 지정된 채널 목록 (전체 또는 특정 채널)

## 실행 내용

### 1. 채널별 커뮤니티 규칙 조사 (웹 서치 + 사이트 방문)

```
각 타겟 채널에 대해 커뮤니티 규칙/가이드라인을 실제로 확인합니다.

[Product Hunt]
- 웹 서치로 "Product Hunt launch rules 2025/2026" 검색
- 웹 서치로 "Product Hunt self promotion policy" 검색
- 사이트 방문으로 현재 런칭 가이드라인 확인
- 수집: 런칭 규칙, 금지 행위, upvote 요청 정책, 재런칭 규칙

[Hacker News]
- 웹 서치로 "Hacker News Show HN guidelines" 검색
- 사이트 방문으로 https://news.ycombinator.com/showhn.html 확인
- 수집: Show HN 규칙, 자기홍보 정책, 커뮤니티 기대 수준

[Reddit — 서브레딧별]
- 각 타겟 서브레딧의 규칙 페이지 직접 방문
- 웹 서치로 "r/{서브레딧} rules self promotion" 검색
- 수집: 셀프프로모 규칙 (10:1 비율 등), 필수 플레어, 금지 항목,
  포스팅 빈도 제한, 계정 나이/카르마 요건, 주간 스레드 활용법

[디스콰이엇]
- 사이트 방문으로 현재 커뮤니티 가이드라인 확인
- 웹 서치로 "디스콰이엇 프로젝트 등록 가이드" 검색
- 수집: 프로젝트 등록 규칙, 메이커로그 작성 가이드, 금지 행위

[네이버 블로그]
- 웹 서치로 "네이버 블로그 저품질 기준 2025/2026" 검색
- 웹 서치로 "네이버 C-Rank D.I.A. 알고리즘 최신" 검색
- 수집: 저품질 판단 기준, 상위 노출 조건, 광고성 글 제한, 최소 글자수

[Twitter/X]
- 웹 서치로 "Twitter/X self promotion rules 2025/2026" 검색
- 수집: 스팸 정책, 해시태그 제한, 자동화 규칙

[Meta (Facebook/Instagram)]
- 웹 서치로 "Facebook group promotion rules" 검색
- 웹 서치로 "Instagram algorithm 2025/2026 best practices" 검색
- 수집: 그룹별 홍보 정책, 알고리즘 선호 콘텐츠 형식, 해시태그 제한

[IndieHackers / Dev.to / GeekNews / OKKY / Velog]
- 각 플랫폼의 커뮤니티 가이드라인 사이트 방문으로 확인
- 웹 서치로 "{플랫폼} posting rules self promotion" 검색
- 수집: 셀프프로모 정책, 콘텐츠 포맷 요구사항, 태그 규칙

[Slack/Discord 커뮤니티]
- 해당 커뮤니티의 채널 설명/규칙 확인 (웹 서치)
- 수집: #self-promotion 채널 존재 여부, 홍보 빈도 제한, 금지 형식
```

### 2. 채널별 최신 인기 콘텐츠 분석 (웹 서치)

```
각 채널에서 최근 인기를 얻은 콘텐츠의 패턴을 조사합니다.

[Product Hunt]
- 웹 서치로 "Product Hunt top products this week/month {카테고리}" 검색
- 사이트 방문으로 최근 #1 데일리 제품 5개 확인
- 분석: 타이틀 패턴, 설명 길이, 이미지 스타일, 메이커 코멘트 길이

[Hacker News]
- 웹 서치로 "Show HN top posts this month" 검색
- 사이트 방문으로 최근 고득점 Show HN 5개 확인
- 분석: 제목 패턴, 본문 길이, 기술 스택 언급 여부, 댓글 반응

[Reddit]
- 각 타겟 서브레딧의 최근 인기글(top week/month) 확인
- 웹 서치로 "r/{서브레딧} top posts this month" 검색
- 분석: 제목 패턴, upvote 수, 댓글 수, 글 유형(질문/공유/토론)
- 특히 "I built..." 유형 중 고득점 포스트 집중 분석

[디스콰이엇]
- 사이트 방문으로 최근 인기 프로젝트 5개 확인
- 분석: 소개 길이, 스크린샷 수, 카테고리, 댓글 반응

[네이버 블로그]
- 타겟 키워드로 네이버 검색 후 상위 5개 블로그 글 확인
- 분석: 글 길이, 이미지 수, 소제목 구조, 키워드 밀도, 태그 수

[Twitter/X]
- 웹 서치로 "{카테고리} launch tweet viral 2025/2026" 검색
- 웹 서치로 "#buildinpublic top tweets" 검색
- 분석: 스레드 길이, 미디어 첨부 여부, 해시태그 수, 참여율

[Meta]
- 웹 서치로 "{카테고리} viral instagram/facebook post" 검색
- 분석: 캐러셀 vs 릴스 vs 단일 이미지 성과 비교

[커뮤니티 (IndieHackers, Dev.to 등)]
- 각 플랫폼의 최근 인기글 확인
- 분석: 제목 패턴, 글 길이, 태그, 반응 수
```

### 3. 채널 건강도 평가

```
각 채널의 현재 활성도와 우리 제품과의 적합도를 평가합니다.

[평가 항목]
1. 활성도: 최근 포스트 빈도, 댓글 수 트렌드 (증가/감소/유지)
2. 타겟 유저 존재: 우리 타겟 유저가 실제로 활동하는지 증거
3. 셀프프로모 수용도: 제품 홍보에 대한 커뮤니티 반응 (호의적/중립/적대적)
4. 경쟁 밀도: 유사 제품이 이미 많이 홍보되고 있는지
5. 우선순위 판정: 높음/중간/낮음 + 근거
```

## 완료 조건

- 모든 타겟 채널의 커뮤니티 규칙 조사 완료
- 각 채널별 최신 인기 콘텐츠 최소 3개 분석
- 채널 우선순위 매트릭스 작성
- 채널별 DO/DON'T 가이드 작성
- `channel-research.md` 파일 생성

## 사용처

- 모든 `write-*` 스킬이 이 파일의 해당 채널 섹션을 참조
- 커뮤니티 규칙을 위반하지 않는 콘텐츠 작성의 기반
- 채널 우선순위로 `/promote` 실행 순서 결정
