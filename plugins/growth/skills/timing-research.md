# Timing Research

최적 포스팅 타이밍, 업계 이벤트, 경쟁사 런칭 일정을 실제 조사합니다.

## 설명

"언제 올릴까"를 감이 아닌 데이터로 결정합니다. 각 채널의 최적 포스팅 시간, 현재 업계 이벤트와 트렌드, 경쟁사 런칭 일정을 웹 서치로 조사하여, 타이밍 충돌을 피하고 최대 노출을 얻을 수 있는 일정을 설계합니다.

> "Product Hunt에서 월요일에 올리면 화요일 런칭에 밀린다. HN에서 금요일에 올리면 주말에 묻힌다. 타이밍이 콘텐츠만큼 중요하다."

## 트리거

- `/promote` 실행 시 `audience-language-research` 다음으로 실행
- `outputs/promotion/audience-language.md` 완료 후

## 입력

- `outputs/promotion/channel-research.md` — 타겟 채널 우선순위
- `outputs/stage-5/market-research.md` — 경쟁사 목록
- 프로젝트 컨텍스트 (제품 카테고리, 타겟 시장)

## 실행 내용

### 1. 채널별 최적 포스팅 시간 조사 (웹 서치)

```
각 채널의 최신 최적 포스팅 시간 데이터를 조사합니다.
(기존 스킬에 하드코딩된 시간이 아닌 최신 데이터 확인)

[Product Hunt]
- 웹 서치로 "best time to launch on Product Hunt 2025/2026" 검색
- 웹 서치로 "Product Hunt launch day of week best" 검색
- 수집: 최적 요일, 시간대, 피해야 할 시기, 시즌별 차이

[Hacker News]
- 웹 서치로 "best time to post Show HN 2025/2026" 검색
- 웹 서치로 "Hacker News traffic pattern by day" 검색
- 수집: 최적 요일/시간, 트래픽 패턴, 뉴스 사이클과의 관계

[Reddit]
- 웹 서치로 "best time to post on reddit {서브레딧}" 검색
- 웹 서치로 "reddit traffic by day of week" 검색
- 수집: 서브레딧별 최적 시간, 시간대 고려 (UTC/EST/KST)

[Twitter/X]
- 웹 서치로 "best time to tweet for tech audience 2025/2026" 검색
- 웹 서치로 "Twitter engagement by hour day" 검색
- 수집: 최적 요일/시간, 타겟 팔로워 활동 시간대

[네이버 블로그]
- 웹 서치로 "네이버 블로그 최적 발행 시간 2025/2026" 검색
- 수집: 최적 발행 시간, 요일별 차이, 네이버 인덱싱 타이밍

[Instagram/Facebook]
- 웹 서치로 "best time to post instagram/facebook 2025/2026" 검색
- 수집: 플랫폼별 최적 시간, 릴스 vs 피드 차이

[이메일]
- 웹 서치로 "best time to send email newsletter 2025/2026" 검색
- 수집: 최적 발송 시간, 요일별 오픈율 차이
```

### 2. 업계 이벤트 & 트렌드 타이밍 조사 (웹 서치)

```
현재/예정된 업계 이벤트를 조사하여 타이밍 기회와 충돌을 파악합니다.

[조사 방법]
- 웹 서치로 "{카테고리} conference event 2025/2026" 검색
- 웹 서치로 "{카테고리} upcoming launch/release" 검색
- 웹 서치로 "tech events calendar {월}" 검색
- 웹 서치로 "Product Hunt upcoming launches" 검색

[수집 항목]
1. 주요 업계 이벤트 (다가오는 것)
   - 이벤트명, 날짜, 관련성
   - 우리 런칭과의 관계 (기회/충돌)

2. 트렌드 타이밍
   - 현재 업계 화제 (우리 제품과 연결 가능한 것)
   - 관련 법/제도 변화 (있으면)
   - 계절성 패턴 (연초 계획, 분기 예산, 연말 정리 등)

3. 뉴스 사이클
   - 현재 테크 뉴스 화제 (타이밍 편승 가능 여부)
   - 피해야 할 기간 (대형 발표, 정치 이벤트 등)
```

### 3. 경쟁사 런칭/활동 일정 조사 (웹 서치)

```
경쟁사의 최근/예정 활동을 조사하여 타이밍 충돌을 방지합니다.

[조사 방법]
- 웹 서치로 "{경쟁사} launch/release 2025/2026" 검색
- 웹 서치로 "{경쟁사} changelog/updates" 검색
- 웹 서치로 "{경쟁사} product hunt" 검색 (PH 런칭 예정 확인)
- 경쟁사 Twitter/블로그에서 최근 활동 확인

[수집 항목]
1. 경쟁사 최근 런칭/업데이트 일정
   - {경쟁사}: {날짜} — {내용}
   
2. 경쟁사 예정 활동 (파악 가능한 것)
   - {경쟁사}: {예정일} — {내용}

3. 타이밍 충돌 분석
   - 같은 날/주에 런칭하면 관심 분산 리스크
   - 경쟁사 런칭 직후는 비교 콘텐츠 올리기 좋은 타이밍
```

### 4. 종합 런칭 타이밍 설계

```
위 조사를 종합하여 최적 일정을 설계합니다.

[설계 항목]
1. D-Day 후보 (3개 제안)
   - 각 후보의 장단점
   - 추천 후보 + 근거

2. 채널별 포스팅 순서
   - 어떤 채널을 먼저 올리고, 어떤 채널을 나중에 올릴지
   - 채널 간 시차 (동시? 순차?)

3. 주간 타임라인
   - D-7 ~ D-Day ~ D+7 상세 일정
   - 각 날짜별 어떤 채널에 어떤 콘텐츠를 올릴지
```
