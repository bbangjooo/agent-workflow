# Wireframes

Step 2.5: 와이어프레임 정의

## 설명

각 화면의 레이아웃과 컴포넌트 배치를 정의하는 스킬입니다. ASCII/텍스트 기반 와이어프레임으로 화면 구조를 시각화합니다.

**플랫폼에 따라 별도의 산출물을 생성합니다.**

## 트리거

- Step 2.4 (Component Spec) 완료 후 자동 실행
- 플랫폼에 맞는 `component-spec-{platform}.md` 파일이 존재할 때

## 입력

- `outputs/stage-2/platform-selection.md` (플랫폼 확인)
- `outputs/stage-2/component-spec-web.md` 또는 `component-spec-mobile.md`
- `outputs/stage-1/screen-structure.md`
- `outputs/stage-1/user-flow.md`

## 실행 내용

### 사전 작업

1. **platform-selection.md 확인** - Web/Mobile/Both 중 어떤 플랫폼인지 확인
2. screen-structure.md에서 화면 목록 추출
3. 각 화면별 필요 컴포넌트 매핑
4. 화면 우선순위 결정 (MVP 핵심 화면부터)

### 플랫폼별 와이어프레임 접근

#### Web 플랫폼
- **반응형 고려**: Desktop을 기준으로 작성하고, Tablet/Mobile breakpoint 동작 설명
- **Breakpoints**: Desktop (>1024px), Tablet (640-1024px), Mobile (<640px)
- **레이아웃**: 그리드 기반, 사이드바, 헤더/푸터 패턴

#### Mobile 플랫폼
- **디바이스 기준**: iPhone/Android 표준 화면 크기 기준
- **네비게이션**: Bottom Tab Bar, Stack Navigation, Drawer
- **플랫폼 가이드라인**: iOS HIG, Material Design 3 패턴 반영
- **제스처**: 스와이프, 풀투리프레시 등 모바일 특화 인터랙션

### 와이어프레임 작성 순서

1. **핵심 화면 먼저**: 사용자가 가장 많이 보는 화면
2. **흐름 순서**: 사용자 여정 순서대로
3. **유사 화면 그룹화**: 비슷한 레이아웃은 한번에

### 질문 가이드

#### Web 플랫폼
1. **레이아웃 선호도**
   - "메인 화면은 어떤 구조를 원하세요? (목록형/카드형/대시보드형)"
   - "사이드바가 필요한가요?"

2. **반응형 동작**
   - "모바일 화면에서는 어떻게 보여야 할까요?"
   - "태블릿에서 컬럼 수를 어떻게 조정할까요?"

#### Mobile 플랫폼
1. **네비게이션 패턴**
   - "하단 탭바를 사용할까요, 아니면 햄버거 메뉴를 사용할까요?"
   - "메인 탭은 몇 개가 적당할까요? (보통 3-5개)"

2. **화면 전환**
   - "이 화면에서 다음 화면으로 어떻게 이동하나요? (푸시/모달/바텀시트)"

### 레이아웃 패턴 가이드

#### Web 레이아웃 패턴
| 패턴 | 용도 | 예시 |
|------|------|------|
| 목록형 | 여러 항목 나열 | 게시글 목록, 채팅 목록 |
| 카드 그리드 | 시각적 항목 | 상품 목록, 갤러리 |
| 대시보드 | 여러 정보 한눈에 | 관리자 페이지, 분석 |
| 상세형 | 단일 항목 상세 | 상품 상세, 프로필 |
| 폼 중심 | 정보 입력 | 회원가입, 설정 |
| 랜딩 | 서비스 소개 | 홈페이지, 마케팅 |

#### Mobile 레이아웃 패턴
| 패턴 | 용도 | 예시 |
|------|------|------|
| 피드형 | 무한 스크롤 콘텐츠 | SNS 피드, 뉴스 |
| 탭 기반 | 메인 섹션 분리 | 홈/검색/프로필 |
| 카드 리스트 | 항목 나열 | 채팅 목록, 알림 |
| 상세 + 액션 | 콘텐츠 + 하단 CTA | 상품 상세, 결제 |
| 폼 시퀀스 | 단계별 입력 | 회원가입, 설문 |
| 모달/바텀시트 | 추가 액션 | 옵션 선택, 필터 |

### 대화 원칙

- 한 화면씩 순차적으로 진행
- ASCII 와이어프레임으로 시각화
- 각 영역에 들어갈 컴포넌트 명시
- **플랫폼에 맞는 패턴 사용** (Web: 반응형, Mobile: 네이티브 패턴)
- "v0에 바로 넣을 수 있는 형태"로 설명 추가 (Web)
- "React Native/Flutter로 바로 구현 가능한 형태"로 설명 추가 (Mobile)
