# Wireframes

Step 2.4: 와이어프레임 정의

## 설명

각 화면의 레이아웃과 컴포넌트 배치를 정의하는 스킬입니다. ASCII/텍스트 기반 와이어프레임으로 화면 구조를 시각화합니다.

## 트리거

- Step 2.3 (Component Spec) 완료 후 자동 실행
- `component-spec.md` 파일이 존재할 때

## 입력

- `outputs/stage-2/component-spec.md`
- `outputs/stage-1/screen-structure.md`
- `outputs/stage-1/user-flow.md`

## 실행 내용

### 사전 작업

1. screen-structure.md에서 화면 목록 추출
2. 각 화면별 필요 컴포넌트 매핑
3. 화면 우선순위 결정 (MVP 핵심 화면부터)

### 와이어프레임 작성 순서

1. **핵심 화면 먼저**: 사용자가 가장 많이 보는 화면
2. **흐름 순서**: 사용자 여정 순서대로
3. **유사 화면 그룹화**: 비슷한 레이아웃은 한번에

### 질문 가이드

1. **레이아웃 선호도**
   - "메인 화면은 어떤 구조를 원하세요? (목록형/카드형/대시보드형)"
   - "모바일 우선인가요, 데스크톱 우선인가요?"

2. **화면별 확인**
   - "이 화면의 와이어프레임이에요. 수정할 부분 있나요?"
   - "이 정보들의 우선순위가 맞나요?"

### 레이아웃 패턴 가이드

| 패턴 | 용도 | 예시 |
|------|------|------|
| 목록형 | 여러 항목 나열 | 게시글 목록, 채팅 목록 |
| 카드 그리드 | 시각적 항목 | 상품 목록, 갤러리 |
| 대시보드 | 여러 정보 한눈에 | 관리자 페이지, 분석 |
| 상세형 | 단일 항목 상세 | 상품 상세, 프로필 |
| 폼 중심 | 정보 입력 | 회원가입, 설정 |
| 랜딩 | 서비스 소개 | 홈페이지, 마케팅 |

### 대화 원칙

- 한 화면씩 순차적으로 진행
- ASCII 와이어프레임으로 시각화
- 각 영역에 들어갈 컴포넌트 명시
- 반응형 고려사항 언급
- "v0에 바로 넣을 수 있는 형태"로 설명 추가
