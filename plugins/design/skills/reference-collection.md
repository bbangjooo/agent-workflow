# Reference Analysis

Step 2.2: 레퍼런스 수집 및 분석

## 설명

만들고자 하는 제품과 유사한 맥락의 **실제 운영 중인 서비스**를 찾아 수집하고, 디자인 요소를 분석하는 스킬입니다. "예쁜 사이트"가 아니라 **타겟 유저와 문제가 유사한 서비스**를 찾는 것이 핵심입니다.

> "AI가 디자인을 못하는 게 아니라, 우리가 AI에게 너무 많은 결정을 위임했다."
> 레퍼런스는 AI에게 구체적인 방향을 제시하는 가장 강력한 도구입니다.

## 트리거

- Step 2.1 (Screen Analysis) 완료 후

## 입력

- `outputs/stage-2/platform-selection.md`

## 실행 내용

### 1. 레퍼런스 탐색

```
사용자에게 질문:
1. "참고하고 싶은 서비스가 이미 있나요?"
2. "어떤 느낌의 서비스를 만들고 싶으세요? (예: Notion처럼 깔끔한, Discord처럼 활기찬)"
3. "경쟁 서비스나 비슷한 서비스를 알고 있나요?"

핵심 원칙:
- "예쁜 사이트"가 아니라 비즈니스 맥락이 유사한 사이트를 찾을 것
- 디자인 포트폴리오 사이트(Dribbble 등)가 아닌 실제 운영 중인 서비스
- 최소 3개, 최대 5개 레퍼런스 수집
```

### 2. 레퍼런스 분석 (4가지 축)

```
각 레퍼런스에 대해 분석:

[레이아웃]
- 전체 구조 (사이드바? 탑바? 풀스크린?)
- 컨텐츠 영역 너비 (narrow/medium/wide)
- 여백 스타일 (넉넉한/타이트한)
- 그리드 시스템 (컬럼 수, 간격)

[색상]
- Primary 색상 계열 (Blue? Green? Purple?)
- 배경 톤 (밝은/어두운/중성)
- 색상 사용량 (미니멀/다채로운)
- 대비 수준 (높은/낮은)

[타이포그래피]
- 폰트 스타일 (산세리프/세리프/모노)
- 글자 크기 범위 (큰 헤딩? 작은 본문?)
- 글자 무게 활용 (볼드 많이? 라이트 중심?)

[그래픽/장식]
- 일러스트레이션 스타일 (있으면)
- 아이콘 스타일 (라인/필드/둘 다)
- 보더/그림자 활용
- 애니메이션/트랜지션 유무
```

### 3. UX 패턴 분석 (screen-analysis 기반)

```
screen-analysis.md의 각 화면별로:
- 같은 목적의 화면을 레퍼런스에서 어떻게 처리하는지
- 정보 위계를 어떻게 시각적으로 표현하는지
- UX 라이팅 패턴 (CTA 문구, 안내 문구 스타일)
- 핵심 액션을 어떻게 유도하는지

→ 와이어프레임(다음 단계)의 직접적 입력이 됨
```

### 4. 비주얼 방향성 도출

```
3-5개 레퍼런스에서 공통으로 나타나는 패턴 정리:
- 공통 레이아웃 패턴
- 공통 색상 톤
- 공통 타이포 스타일
- 브랜드 키워드 3-5개 (예: 심플한, 신뢰감, 친근한)
- 분위기 한줄 정의
- 차별화 포인트

→ 와이어프레임 + 디자인 토큰 단계의 입력이 됨
```

### 내부 레퍼런스 라이브러리

`references/` 디렉토리 아래의 **모든 하위 폴더와 .md 파일**이 내부 레퍼런스입니다. 새 프로젝트 시작 시 `references/` 전체를 탐색하여 유사한 맥락의 레퍼런스가 있는지 먼저 확인하세요.

```
탐색 방법:
1. `references/` 하위 디렉토리 목록을 확인 (Glob: references/*/)
2. 각 카테고리 내 .md 파일을 확인 (Glob: references/**/*.md)
3. 프로젝트 맥락과 유사한 레퍼런스를 선별하여 읽고 분석
```

> **주의**: 카테고리는 수시로 추가될 수 있으므로, 특정 폴더명을 하드코딩하지 말고 항상 디렉토리를 동적으로 탐색할 것.
