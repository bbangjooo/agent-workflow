# Screen Analysis

Step 2.1: 기획 분석 — 화면별 목적/유저상황/핵심기능 정리

## 설명

PRD와 화면 구조를 분석하여 **각 화면이 왜 존재하는지, 유저가 어떤 상황에서 보는지, 무엇을 잘 보여줘야 하는지**를 정리하는 스킬입니다. 디자인의 모든 결정(레이아웃, 정보 위계, 강조 요소)은 이 분석에 기반합니다.

> 디자이너가 화면을 그리기 전에 가장 먼저 하는 일: "이 화면의 목적이 뭐야?"

## 실행 내용

### 각 화면에 대해 분석

```
1. 화면 목적 (Purpose)
   - 이 화면은 왜 존재하는가?
   - 유저가 이 화면에서 달성해야 할 것은?
   - 예: "상품 목록 → 유저가 원하는 상품을 빠르게 찾아 상세로 이동"

2. 유저 상황 (User Context)
   - 어떤 상태의 유저가 이 화면을 보는가?
   - 유저의 감정/의도는? (탐색 중? 결정 직전? 완료 후?)
   - 예: "결제 완료 → 안도감, '잘 됐나?' 확인 필요"

3. 핵심 정보 위계 (Information Hierarchy)
   - 가장 먼저 눈에 들어와야 할 정보 (1순위)
   - 보조 정보 (2순위)
   - 있으면 좋지만 필수는 아닌 정보 (3순위)
   - 예: "상품 상세 → 1순위: 상품 이미지/가격, 2순위: 설명/리뷰, 3순위: 관련 상품"

4. 핵심 액션 (Primary Action)
   - 이 화면에서 유저가 해야 할 주요 액션
   - CTA(Call to Action) 무엇인지
   - 예: "회원가입 → CTA: '시작하기' 버튼"

5. 진입/이탈 경로 (Flow Context)
   - 어디서 이 화면으로 왔는가?
   - 이 화면 이후 어디로 가는가?
   - 이전/이후 화면과의 맥락 연결
```

### 질문 가이드

```
1. "이 화면에서 유저에게 가장 중요한 정보가 뭐라고 생각하세요?"
2. "유저가 이 화면에서 어떤 행동을 하길 원하세요?"
3. "이 화면을 볼 때 유저의 심리 상태는 어떨까요?"
```
