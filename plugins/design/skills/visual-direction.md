# Visual Direction

Step 2.2: 비주얼 방향성 정의

## 설명

제품의 시각적 톤앤매너를 결정하는 스킬입니다. 브랜드 키워드, 분위기, 참고 서비스를 정의하여 이후 디자인 시스템의 기초를 만듭니다.

## 트리거

- Step 2.1 (Color Palette) 완료 후 실행
- `outputs/stage-2/color-palette.md` 파일이 존재할 때

## 입력

- `outputs/stage-2/color-palette.md`
- `outputs/stage-0/idea-brief.md`
- `outputs/stage-1/prd.md`
- `outputs/stage-1/screen-structure.md`

## 실행 내용

### 사전 작업

1. PRD에서 타겟 사용자 정보 추출
2. 제품의 핵심 가치 파악
3. 참고할 수 있는 유사 서비스 후보 준비

### 질문 가이드

1. **전체 분위기**
   - "어떤 느낌의 서비스를 원하세요? (모던/미니멀/친근/전문적/활기찬)"
   - "타겟 사용자가 좋아할 것 같은 분위기는요?"

2. **참고 서비스**
   - "디자인적으로 참고하고 싶은 서비스가 있나요?"
   - "이런 느낌은 어때요? (비슷한 서비스 2-3개 제안)"

3. **피해야 할 스타일**
   - "절대 피하고 싶은 디자인 스타일이 있나요?"
   - "예를 들어 '너무 복잡한', '어두운', '유치한' 같은 키워드로 말씀해주세요"

4. **브랜드 키워드**
   - "서비스를 3-5개의 단어로 표현한다면요?"
   - "예: 심플한, 신뢰감 있는, 젊은, 프리미엄..."

### 분위기 옵션 가이드

| 분위기 | 설명 | 참고 서비스 예시 |
|--------|------|-----------------|
| 모던/미니멀 | 깔끔, 여백 활용, 제한된 색상 | Apple, Notion, Linear |
| 친근/캐주얼 | 둥근 모서리, 밝은 색상, 일러스트 | Duolingo, Slack, Figma |
| 전문적/신뢰 | 직선적, 차분한 색상, 정돈된 레이아웃 | Stripe, Intercom, HubSpot |
| 활기찬/볼드 | 강렬한 색상, 큰 타이포, 대비 | Spotify, Discord, Canva |
| 프리미엄/럭셔리 | 어두운 배경, 골드/화이트, 세리프 폰트 | Amex, Porsche, Rolex |

### 대화 원칙

- 구체적인 예시 서비스를 언급하며 설명
- "이 서비스 아세요?" 같은 질문으로 공감대 형성
- 사용자가 모르면 스크린샷 설명으로 대체
- 너무 많은 선택지를 한번에 주지 않기
