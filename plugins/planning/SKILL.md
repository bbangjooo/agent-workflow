# Planning (기획)

1인 창업자의 아이디어 브리프를 실행 가능한 PRD(Product Requirements Document)로 발전시키는 워크플로우 플러그인입니다.

## 개요

이 플러그인은 Stage 0(아이디어 고도화)에서 생성된 아이디어 브리프를 입력으로 받아, 디자인과 개발에 바로 활용할 수 있는 기획 문서를 생성합니다.

## 사전 조건

- Stage 0 완료 (`/ideate`)
- `outputs/stage-0/idea-brief.md` 파일 존재

## 사용 방법

```
/plan
```

이 커맨드 하나로 Planning Coach 에이전트가 전체 워크플로우를 안내합니다.

## 워크플로우

├─────────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────────┤
│  Step 1.2: User Stories (사용자 스토리)                     │

├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 1.3: User Flow (사용자 흐름)                          │
│  스킬: user-flow                                            │
│  산출물: user-flow.md                                       │
├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 1.4: Screen Structure (화면 구조)                     │

## 주요 컴포넌트

### Command

- **/plan**: Stage 1 전체 워크플로우 시작 (유일한 진입점)

### Agent

- **Planning Coach**: 체계적이고 실용적인 기획 코치. Step 순서를 관리하며 사용자와 대화를 통해 PRD를 완성합니다.

### Skills

| 스킬 | Step | 설명 |
|------|------|------|
| feature-prioritization | 1.1 | MVP 기능 우선순위 정리 |
| user-stories | 1.2 | 사용자 스토리 작성 |
| user-flow | 1.3 | 사용자 흐름 정의 |
| screen-structure | 1.4 | 화면 구조 정의 |
| prd-generation | 1.5 | 최종 PRD 문서 생성 |

## 산출물

모든 산출물은 `outputs/stage-1/` 디렉토리에 저장됩니다:

| 파일명 | Step | 설명 |
|--------|------|------|
| feature-priority.md | 1.1 | 기능 우선순위 (P0~P3) |
| user-stories.md | 1.2 | 사용자 스토리 및 인수 조건 |
| user-flow.md | 1.3 | 사용자 흐름도 |
| screen-structure.md | 1.4 | 화면 목록 및 구조 |
| **prd.md** | 1.5 | **최종 PRD** |
