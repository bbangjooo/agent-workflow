# Screen Structure

Step 1.4: 화면 구조 정의

## 설명

User Flow에서 도출된 화면들을 정리하고, 각 화면의 목적과 주요 구성요소를 정의하는 스킬입니다.

## 트리거

- Step 1.3 (User Flow) 완료 후 자동 실행
- `user-flow.md` 파일이 존재할 때

## 입력

- `outputs/stage-1/feature-priority.md`
- `outputs/stage-1/user-stories.md`
- `outputs/stage-1/user-flow.md`

## 실행 내용

### 사전 작업

1. user-flow.md에서 모든 화면 추출
2. 중복 제거 및 화면 목록 정리
3. 각 화면의 역할 정의

### 질문 가이드

1. **화면 목적**
   - "이 화면에서 사용자가 뭘 하나요?"
   - "이 화면의 가장 중요한 역할은 뭔가요?"

2. **화면 구성요소**
   - "이 화면에 어떤 정보가 보여야 하나요?"
   - "어떤 버튼이나 입력창이 필요하나요?"

3. **화면 간 관계**
   - "이 화면에서 어디로 갈 수 있나요?"
   - "메뉴나 네비게이션은 어떻게 구성할까요?"

### 화면 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 랜딩/홈 | 서비스 소개, 진입점 | 랜딩페이지, 메인 대시보드 |
| 인증 | 로그인, 회원가입 | 로그인, 가입, 비밀번호 찾기 |
| 목록 | 여러 항목 표시 | 상품 목록, 게시글 목록 |
| 상세 | 단일 항목 상세 정보 | 상품 상세, 게시글 상세 |
| 폼 | 정보 입력 | 글쓰기, 설정, 결제 |
| 결과 | 액션 결과 표시 | 완료, 에러, 빈 상태 |
