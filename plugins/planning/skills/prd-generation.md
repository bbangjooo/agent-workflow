# PRD Generation

Step 1.5: PRD 생성

## 설명

이전 단계의 모든 산출물을 하나의 Product Requirements Document(PRD)로 통합하는 스킬입니다. Stage 2(디자인)로 넘어가기 위한 핸드오프 문서를 생성합니다.

## 트리거

- Step 1.4 (Screen Structure) 완료 후 자동 실행
- 모든 이전 산출물이 존재할 때

## 입력

- `outputs/stage-0/idea-brief.md`
- `outputs/stage-1/feature-priority.md`
- `outputs/stage-1/user-stories.md`
- `outputs/stage-1/user-flow.md`
- `outputs/stage-1/screen-structure.md`

## 실행 내용

### 작업 순서

1. 모든 이전 산출물 읽기
2. 핵심 내용 추출 및 정리
3. PRD 문서 초안 작성
4. 사용자에게 검토 요청
5. 피드백 반영 후 최종 확정

### 사용자 확인 요청

"모든 기획 내용을 PRD로 정리했어요. 확인해주시겠어요?

**프로젝트**: {프로젝트명}
**MVP 기능**: {N}개
**화면 수**: {N}개

{PRD 주요 내용 요약}

수정할 부분이 있으면 말씀해주세요. 괜찮으시면 이 PRD를 기반으로 **디자인 단계**로 넘어갈 수 있어요!"

## 산출물

`outputs/stage-1/prd.md` (Stage 1 최종 산출물)

```markdown
# Product Requirements Document (PRD)

## 메타데이터
- Stage: 1 (완료)
- 생성일시: {현재 시간}
- 상태: final
- 다음 Stage: Stage 2 - 디자인

---

## 1. 프로젝트 개요

### 프로젝트명
{프로젝트명}

### 한 줄 설명
{서비스 요약}

### 타겟 사용자
{주요 사용자 유형}

### 핵심 문제
{해결하려는 문제}

### 핵심 가치 제안
{사용자가 얻는 가치}

---

## 2. MVP 범위

### 포함 기능 (In Scope)

#### P0: 핵심 필수
| 기능 | 설명 |
|------|------|
| {기능} | {설명} |

#### P1: 중요
| 기능 | 설명 |
|------|------|
| {기능} | {설명} |

### 제외 기능 (Out of Scope)
- {나중에 할 기능 1}
- {나중에 할 기능 2}

---

## 3. 사용자 스토리

### 핵심 스토리

#### {기능명}
> {사용자}로서, {행동}을 하고 싶다.
> 왜냐하면 {가치} 때문이다.

**인수 조건**
- [ ] {조건 1}
- [ ] {조건 2}

---

## 4. 사용자 흐름

### 온보딩 흐름
```
[시작] → [화면1] → [화면2] → [완료]
```

### 핵심 기능 흐름
```
[시작] → [화면1] → [화면2] → [완료]
```

---

## 5. 화면 구조

### 화면 목록
| ID | 화면명 | 목적 |
|----|--------|------|

### 네비게이션
```
[메인]
├── {메뉴1}
├── {메뉴2}
└── {메뉴3}
```

### 주요 화면 상세

#### S01: {화면명}
- 목적: {목적}
- 구성요소: {요소}
- 연결: {다음 화면}

---

## 6. 기술 고려사항

### 플랫폼
- {웹/앱/둘다}

### 제약사항
- {기술적 제약 또는 요구사항}

---

## 7. 다음 단계

Stage 2 (디자인)에서 진행할 내용:
- [ ] 디자인 시스템 정의
- [ ] 와이어프레임 작성
- [ ] UI 디자인

---

## 승인

- [ ] 이 PRD 내용을 확인하고 승인합니다
- 승인일: {날짜}
```

## 완료 조건

- 모든 이전 산출물이 통합됨
- 사용자가 PRD 내용을 확인함
- `prd.md` 파일이 생성됨
- Stage 1 완료 상태로 전환

## 완료 안내

"축하해요! Stage 1 '기획'이 완료됐습니다!

**PRD 문서**가 생성되었어요. 이 문서를 기반으로 다음 단계인 **Stage 2: 디자인**을 시작할 수 있어요.

준비되시면 `/design` 커맨드로 디자인 단계를 시작하세요!"
