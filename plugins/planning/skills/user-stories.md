# User Stories

Step 1.2: 사용자 스토리 작성

## 설명

MVP 기능들을 사용자 관점의 스토리로 변환하는 스킬입니다. 개발자가 아닌 사람도 이해할 수 있는 형태로 기능을 정의합니다.

## 트리거

- Step 1.1 (Feature Prioritization) 완료 후 자동 실행
- `feature-priority.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/idea-brief.md`
- `outputs/stage-1/feature-priority.md`

## 실행 내용

### 사전 작업

1. feature-priority.md에서 P0, P1 기능 추출
2. 각 기능에 대해 사용자 스토리 형식으로 변환

### 사용자 스토리 형식

```
[사용자 유형]으로서,
[어떤 행동]을 하고 싶다.
왜냐하면 [얻고 싶은 가치] 때문이다.
```

또는 간단히:
```
As a [user type], I want to [action], so that [benefit].
```

### 질문 가이드

1. **사용자 유형 확인**
   - "이 기능을 주로 누가 사용하나요?"
   - "로그인한 사용자? 비회원? 관리자?"

2. **행동 구체화**
   - "사용자가 정확히 뭘 하고 싶어하는 거예요?"
   - "어떤 버튼을 누르거나 어떤 정보를 입력하나요?"

3. **가치 확인**
   - "이걸 하면 사용자가 뭘 얻나요?"
   - "왜 이 기능이 사용자에게 중요한가요?"

### 예시

| 기능 | 사용자 스토리 |
|------|-------------|
| 회원가입 | 신규 사용자로서, 이메일로 계정을 만들고 싶다. 왜냐하면 서비스를 이용하고 싶기 때문이다. |
| 게시글 작성 | 회원으로서, 글을 작성하고 싶다. 왜냐하면 내 생각을 공유하고 싶기 때문이다. |

- 사용자가 스토리 내용에 동의함
