# Feature Prioritization

Step 1.1: 기능 우선순위 정리

## 설명

idea-brief에서 정의된 MVP 기능들의 우선순위를 정리하고, Must-have와 Nice-to-have를 구분하는 스킬입니다.

## 트리거

- Stage 1 워크플로우 시작 시 첫 번째로 실행
- `outputs/stage-0/idea-brief.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/idea-brief.md`

## 실행 내용

### 사전 작업

1. idea-brief.md에서 MVP 기능 목록 추출
2. 사용자에게 각 기능 확인

### 질문 가이드

1. **필수 vs 선택 구분**
   - "이 기능이 없으면 서비스가 아예 안 되나요, 아니면 불편한 정도인가요?"
   - "첫 번째 사용자에게 꼭 보여줘야 하는 기능인가요?"

2. **우선순위 결정**
   - "이 기능들 중 가장 먼저 만들어야 하는 건 뭘까요?"
   - "만약 하나만 만들 수 있다면 어떤 걸 선택하시겠어요?"

3. **범위 조정**
   - "이 기능은 처음엔 간단하게 만들고 나중에 개선해도 될까요?"
   - "이건 사용자가 많아진 다음에 추가해도 괜찮을 것 같은데, 어떻게 생각하세요?"

### 우선순위 프레임워크

| 구분 | 설명 | 예시 |
|------|------|------|
| P0 (Must) | 이게 없으면 서비스 자체가 불가능 | 로그인, 핵심 기능 |
| P1 (Should) | 없으면 많이 불편하지만 돌아가긴 함 | 알림, 검색 |
| P2 (Could) | 있으면 좋지만 없어도 됨 | 다크모드, 공유 |
| P3 (Later) | 나중에 추가할 기능 | 고급 분석, 팀 기능 |

## 산출물

`outputs/stage-1/feature-priority.md`

```markdown
## 메타데이터
- Stage: 1

- 생성일시: {현재 시간}
- 상태: draft

## MVP 기능 (Must-have)

### P0: 핵심 필수
| 기능 | 설명 | 이유 |
|------|------|------|
| {기능명} | {설명} | {왜 필수인지} |

### P1: 중요
| 기능 | 설명 | 이유 |
|------|------|------|
| {기능명} | {설명} | {왜 중요한지} |

## 추후 기능 (Nice-to-have)

### P2: 있으면 좋음
| 기능 | 설명 | 언제 추가할지 |
|------|------|-------------|
| {기능명} | {설명} | {조건/시기} |

### P3: 나중에
| 기능 | 설명 |
|------|------|
| {기능명} | {설명} |

## MVP 범위 요약

- 총 MVP 기능 수: {N}개
- P0 기능: {N}개
- P1 기능: {N}개

## 결정 근거

{왜 이렇게 우선순위를 정했는지 요약}
```

## 완료 조건

- 모든 기능이 P0~P3으로 분류됨
- MVP 범위가 명확히 정의됨
- 사용자가 우선순위에 동의함
- `feature-priority.md` 파일이 생성됨
