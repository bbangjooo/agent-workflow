# User Flow

Step 1.3: 사용자 흐름 정의

## 설명

사용자가 서비스를 이용하는 핵심 경로를 단계별로 정의하는 스킬입니다. 화면 간 이동과 주요 액션을 시각화합니다.

## 트리거

- Step 1.2 (User Stories) 완료 후 자동 실행
- `user-stories.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/idea-brief.md`
- `outputs/stage-1/feature-priority.md`
- `outputs/stage-1/user-stories.md`

## 실행 내용

### 핵심 흐름 식별

서비스의 핵심 사용자 여정을 2~3개 정의합니다:

1. **온보딩 흐름**: 첫 방문 → 가입 → 첫 사용
2. **핵심 기능 흐름**: 로그인 → 핵심 기능 사용 → 결과 확인
3. **재방문 흐름**: 재방문 → 이전 작업 이어하기

### 질문 가이드

1. **시작점 확인**
   - "사용자가 처음 서비스에 들어오면 뭘 보게 되나요?"
   - "로그인이 필요한가요, 아니면 바로 사용할 수 있나요?"

2. **핵심 경로**
   - "사용자의 목표를 달성하려면 어떤 단계를 거쳐야 하나요?"
   - "각 단계에서 사용자가 뭘 하나요?"

3. **분기점**
   - "여기서 사용자가 선택할 수 있는 옵션이 있나요?"
   - "실패하면 어디로 가나요?"

### 흐름도 표기법

```
[시작] → (행동) → [화면] → (행동) → [결과]

예시:
[랜딩페이지] → (회원가입 클릭) → [가입폼] → (정보입력) → [환영화면]
```
