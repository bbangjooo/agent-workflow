# Stage 0: Ideation (아이디어 고도화)

1인 창업자의 막연한 아이디어를 제품화 가능한 수준까지 구체화하는 워크플로우 플러그인입니다.

## 개요

이 플러그인은 아이디어만 있고 구체적인 기획이 없는 1인 창업자를 위해 설계되었습니다. 5단계의 체계적인 프로세스를 통해 아이디어를 검증하고 구체화합니다.

## 사용 방법

```
/ideate
```

이 커맨드 하나로 Idea Coach 에이전트가 전체 워크플로우를 안내합니다.

## 워크플로우

```
/ideate 실행
    ↓
Idea Coach 에이전트가 대화 주도
    ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 0.1: Idea Capture (아이디어 수집)                      │
│  스킬: idea-capture                                         │
│  산출물: raw-idea.md                                        │
├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 0.2: Problem Definition (문제 정의)                   │
│  스킬: problem-definition                                   │
│  산출물: problem-statement.md                               │
├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 0.3: Solution Framing (솔루션 구체화)                 │
│  스킬: solution-framing                                     │
│  산출물: solution-outline.md                                │
├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 0.4: Idea Validation (검증 및 피드백)                 │
│  스킬: idea-validation                                      │
│  산출물: validation-report.md                               │
├─────────────────────────────────────────────────────────────┤
│                           ↓                                 │
├─────────────────────────────────────────────────────────────┤
│  Step 0.5: Brief Generation (브리프 생성)                   │
│  스킬: brief-generation                                     │
│  산출물: idea-brief.md (최종)                               │
└─────────────────────────────────────────────────────────────┘
```

## 주요 컴포넌트

### Command

- **/ideate**: Stage 0 전체 워크플로우 시작 (유일한 진입점)

### Agent

- **Idea Coach**: 친근하고 체계적인 아이디어 코치. Step 순서를 관리하며 사용자와 대화를 통해 아이디어를 구체화합니다.

### Skills

| 스킬 | Step | 설명 |
|------|------|------|
| idea-capture | 0.1 | 아이디어 수집 및 기록 |
| problem-definition | 0.2 | 문제 정의 및 타겟 사용자 파악 |
| solution-framing | 0.3 | 솔루션 구체화 및 MVP 정의 |
| idea-validation | 0.4 | 시장/기술 검증 및 리스크 분석 |
| brief-generation | 0.5 | 최종 브리프 문서 생성 |

## 산출물

모든 산출물은 `outputs/stage-0/` 디렉토리에 저장됩니다:

| 파일명 | Step | 설명 |
|--------|------|------|
| raw-idea.md | 0.1 | 원본 아이디어 기록 |
| problem-statement.md | 0.2 | 문제 정의서 |
| solution-outline.md | 0.3 | 솔루션 개요 |
| validation-report.md | 0.4 | 검증 리포트 |
| **idea-brief.md** | 0.5 | **최종 아이디어 브리프** |
