# Idea Capture

Step 0.1: 아이디어 수집

## 설명

사용자의 머릿속에 있는 아이디어를 있는 그대로 끌어내는 스킬입니다. 판단이나 평가 없이 자유롭게 표현하도록 유도합니다.

## 트리거

- Stage 0 워크플로우 시작 시 첫 번째로 실행
- 에이전트가 아이디어 수집이 필요하다고 판단할 때

## 입력

- 사용자의 자유로운 아이디어 설명

## 실행 내용

### 질문 가이드

1. "어떤 서비스나 제품을 만들고 싶으신가요? 편하게 말씀해주세요."
2. "이 아이디어는 어떻게 떠오르게 되셨어요?"
3. "이걸 사용할 사람은 누구일까요?"
4. "한 문장으로 표현한다면 뭐라고 할 수 있을까요?"

### 대화 원칙

- 열린 질문 사용
- "그렇군요", "흥미롭네요" 등 공감 표현
- 모호해도 괜찮다고 안심시키기
- 추가 설명이 필요하면 부드럽게 요청

## 산출물

`outputs/stage-0/raw-idea.md`

```markdown
# Raw Idea

## 메타데이터
- Stage: 0
- Step: 0.1 - 아이디어 수집
- 생성일시: {현재 시간}
- 상태: draft

## 원본 아이디어

{사용자가 설명한 아이디어를 있는 그대로 정리}

## 핵심 키워드

- {키워드 1}
- {키워드 2}

## 대상 사용자 (초기 생각)

{사용자가 언급한 타겟 사용자}

## 한 줄 요약

{사용자의 아이디어를 한 문장으로}
```

## 완료 조건

- 아이디어의 핵심이 파악됨
- 대상 사용자가 대략적으로 식별됨
- 한 줄 요약이 가능함
- `raw-idea.md` 파일이 생성됨
