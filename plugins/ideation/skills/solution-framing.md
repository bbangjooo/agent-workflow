# Solution Framing

Step 0.3: 솔루션 구체화

## 설명

문제에 대한 해결책을 구체적으로 정의하는 스킬입니다. 핵심 기능, MVP 범위, 차별점을 명확히 합니다.

## 트리거

- Step 0.2 (Problem Definition) 완료 후 자동 실행
- `problem-statement.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/raw-idea.md`
- `outputs/stage-0/problem-statement.md`
- 사용자와의 추가 대화

## 실행 내용

### 질문 가이드

1. **핵심 기능 도출**
   - "이 문제를 해결하기 위해 꼭 필요한 기능 하나만 고른다면?"
   - "사용자가 처음 앱을 열었을 때 뭘 할 수 있어야 할까요?"
   - "없으면 안 되는 기능 vs 있으면 좋은 기능을 구분해볼까요?"

2. **차별점 정의**
   - "기존 대안들과 가장 다른 점은 뭘까요?"
   - "왜 사용자가 기존 것 대신 이걸 쓸까요?"

3. **사용자 여정**
   - "사용자가 처음 서비스를 알게 되는 순간부터 문제가 해결되는 순간까지를 상상해볼까요?"

4. **형태 결정**
   - "웹, 앱, 둘 다 중 어떤 형태가 좋을까요?"

## 산출물

`outputs/stage-0/solution-outline.md`

## 솔루션 개요

### 한 줄 정의
{솔루션을 한 문장으로}

{사용자가 얻는 핵심 가치}

## 핵심 기능 (MVP)

| 우선순위 | 기능 | 설명 | 필수 여부 |
|---------|------|------|----------|
| 1 | {기능명} | {설명} | 필수 |
| 2 | {기능명} | {설명} | 필수 |
| 3 | {기능명} | {설명} | 선택 |

## 차별점

### 핵심 차별화 포인트
- {차별점 1}
- {차별점 2}

## 완료 조건

- MVP 범위가 명확함
- 차별점이 식별됨
- 제품 형태가 결정됨
- `solution-outline.md` 파일이 생성됨
