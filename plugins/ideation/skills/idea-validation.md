# Idea Validation

Step 0.4: 검증 및 피드백

## 설명

아이디어의 실현 가능성과 시장성을 검토하는 스킬입니다. 경쟁사 분석, 기술적 타당성, 리스크를 평가합니다.

## 트리거

- Step 0.3 (Solution Framing) 완료 후 자동 실행
- `solution-outline.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/raw-idea.md`
- `outputs/stage-0/problem-statement.md`
- `outputs/stage-0/solution-outline.md`

## 실행 내용

### 검증 영역

1. **시장 검증**
   - 유사 서비스 탐색
   - 시장 규모 추정
   - 타이밍 적절성 평가

2. **기술 검증**
   - 현재 기술로 구현 가능한가?
   - 1인 창업자가 만들 수 있는 수준인가?
   - 외부 의존성은?

3. **비즈니스 모델 검토**
   - 수익화 가능성
   - 무료/유료 전략

4. **리스크 분석**
   - 주요 리스크 식별
   - 실패 시나리오

### 사용자 대화

- "비슷한 서비스로 {경쟁사}가 있는데, 알고 계셨나요?"
- "이 부분이 기술적으로 도전적일 수 있어요. 어떻게 생각하세요?"
- "수익 모델은 어떻게 생각하고 계신가요?"
