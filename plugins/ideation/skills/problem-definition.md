# Problem Definition

Step 0.2: 문제 정의

## 설명

아이디어가 해결하려는 문제를 명확하게 정의하는 스킬입니다. 타겟 사용자와 그들이 겪는 구체적인 문제를 파악합니다.

## 트리거

- Step 0.1 (Idea Capture) 완료 후 자동 실행
- `raw-idea.md` 파일이 존재할 때

## 입력

- `outputs/stage-0/raw-idea.md`
- 사용자와의 추가 대화

## 실행 내용

### 질문 가이드

1. **타겟 사용자 심화**
   - "이 서비스를 가장 필요로 하는 사람은 구체적으로 누구일까요?"
   - "그 사람의 하루를 상상해보면, 언제 이 문제를 겪을까요?"

2. **문제 구체화**
   - "그 사람이 지금 겪고 있는 가장 큰 불편함은 뭘까요?"
   - "이 문제가 해결 안 되면 어떤 일이 생기나요?"

3. **기존 대안 파악**
   - "지금은 이 문제를 어떻게 해결하고 있나요?"
   - "기존 방법의 가장 큰 단점은 뭘까요?"

4. **문제 검증**
   - "이 문제가 얼마나 자주 발생하나요?"
   - "이 문제를 겪는 사람이 많을까요?"

## 완료 조건

- 타겟 사용자가 구체적으로 정의됨
- 핵심 문제가 한 문장으로 표현됨
- 기존 대안과 그 한계가 파악됨
- `problem-statement.md` 파일이 생성됨
