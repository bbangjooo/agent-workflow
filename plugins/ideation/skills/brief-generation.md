# Brief Generation

Step 0.5: 아이디어 브리프 생성

## 설명

이전 단계의 모든 산출물을 하나의 종합 문서로 통합하는 스킬입니다. Stage 1(기획)으로 넘어가기 위한 핸드오프 문서를 생성합니다.

## 트리거

- Step 0.4 (Idea Validation) 완료 후 자동 실행
- 모든 이전 산출물이 존재할 때

## 입력

- `outputs/stage-0/raw-idea.md`
- `outputs/stage-0/problem-statement.md`
- `outputs/stage-0/solution-outline.md`
- `outputs/stage-0/validation-report.md`

## 실행 내용

### 작업 순서

1. 모든 이전 산출물 읽기
2. 핵심 내용 추출 및 정리
3. 브리프 문서 초안 작성
4. 사용자에게 검토 요청
5. 피드백 반영 후 최종 확정
6. **목표 추출 (`extract-goals.md` 호출)** (NEW)

### 사용자 확인 요청

"지금까지 논의한 내용을 아이디어 브리프로 정리했어요. 한번 확인해주시겠어요?

{브리프 요약}

수정할 부분이 있으면 말씀해주세요. 괜찮으시면 이 브리프를 기반으로 다음 단계인 '기획'으로 넘어갈 수 있어요!"

## 산출물

`outputs/stage-0/idea-brief.md` (Stage 0 최종 산출물)

```markdown
# Idea Brief

## 메타데이터
- Stage: 0 (완료)
- 생성일시: {현재 시간}
- 상태: final
- 다음 Stage: Stage 1 - 기획

---

## 1. 프로젝트 개요

### 프로젝트명
{프로젝트 이름 또는 코드네임}

### 한 줄 설명
{서비스를 한 문장으로}

### 제품 형태
- 플랫폼: {웹/앱/둘다}
- 타입: {서비스 유형}

---

## 2. 문제 정의

### 타겟 사용자
{누구를 위한 서비스인가}

### 핵심 문제
{어떤 문제를 해결하는가}

### 문제 가설
"{사용자}는 {상황}에서 {문제}를 겪고 있다"

---

## 3. 솔루션

### 핵심 가치 제안
{사용자가 얻는 핵심 가치}

### MVP 기능 목록
1. {필수 기능 1}
2. {필수 기능 2}
3. {필수 기능 3}

---

## 4. 검증 요약

### 시장 기회
{시장 분석 요약}

### 주요 경쟁사
- {경쟁사 1}: {간단한 설명}

### 리스크 및 대응
| 리스크 | 대응 방안 |
|--------|----------|
| {리스크} | {대응} |

### 실행 가능성 점수
{종합 점수} / 5.0

---

## 5. 다음 단계

Stage 1 (기획)에서 진행할 내용:
- [ ] 상세 PRD 작성
- [ ] 사용자 스토리 정의
- [ ] 기능 명세서 작성

---

## 승인

- [ ] 이 브리프 내용을 확인하고 승인합니다
- 승인일: {날짜}
```

## 완료 조건

- 모든 이전 산출물이 통합됨
- 사용자가 브리프 내용을 확인함
- `idea-brief.md` 파일이 생성됨
- **목표(Goals)가 추출되어 state.json에 등록됨** (NEW)
- Stage 0 완료 상태로 전환
