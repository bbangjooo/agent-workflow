# /reflect

Stage 회고 커맨드 - 학습과 개선점을 수집합니다

## 사용법

```
/reflect              # 현재 또는 마지막 완료된 Stage 회고
/reflect ideation     # 특정 Stage 회고
/reflect planning
/reflect design
/reflect development
/reflect deployment
```

## 설명

각 Stage를 완료한 후, 무엇을 배웠고 무엇을 개선할 수 있는지 돌아보는 시간을 갖습니다. 수집된 피드백은 다음 이터레이션이나 새 버전에서 활용됩니다.

## 트리거

- Stage 완료 후 자동 제안
- 사용자가 직접 호출
- `/build-project` 완료 후 전체 회고

## 실행 스킬 순서

1. `gather-learnings.md` - 학습점 수집
2. `identify-improvements.md` - 개선점 도출
3. `suggest-next-action.md` - 다음 액션 제안

## Agent

- `feedback-coach.md` 사용

## 입력

- 대상 Stage (선택적, 없으면 현재/마지막 Stage)
- 해당 Stage의 산출물들

## 산출물

`outputs/feedback/stage-{n}-reflection.md`

```markdown
# Stage {N} Reflection

## 메타데이터

- 회고일: {날짜}
- 소요시간: {시간} (추정)

---

## 1. 잘된 점 (What Went Well)

### {항목 1}

- 결과: {어떤 좋은 결과가 있었나}
- 인사이트: {왜 잘 됐는지}

### {항목 2}
...

## 2. 개선할 점 (What Could Be Improved)

### {항목 1}
- 상황: {구체적 상황}
- 문제: {어떤 어려움이 있었나}
- 제안: {다음에 어떻게 할 것인가}

### {항목 2}
...

---

## 3. 예상과 달랐던 점 (Surprises)

- {놀라웠던 점 2}
```

## 회고 진행 방식

### 1단계: 준비

```
"Stage {N}을 마치셨군요!

잠깐 돌아보는 시간을 가져볼까요?
5-10분이면 충분해요. 나중에 큰 도움이 될 거예요."
```

### 2단계: 잘된 점

```
"먼저 잘 된 점부터 볼까요?

이번 Stage에서 특히 잘 됐다고 느낀 순간이 있나요?
예상보다 빠르게 진행됐거나, 만족스러운 결과가 나온 부분이요."
```

### 3단계: 개선점

```
"이번엔 개선점을 찾아볼게요.

다음에 이 Stage를 다시 한다면, 다르게 하고 싶은 게 있나요?
막혔던 부분이나 시간이 오래 걸린 부분도 좋아요."
```

### 4단계: 놀라운 점

```
"예상과 달랐던 점이 있었나요?

'어? 이건 생각 못 했네' 싶었던 순간이요.
좋은 놀라움이든, 아쉬운 놀라움이든 상관없어요."
```

### 5단계: 다음 시도

```
"마지막으로, 다음에 시도해보고 싶은 게 있나요?

새로운 방법, 도구, 접근법... 뭐든 좋아요."
```

### 6단계: 정리 및 제안

```
"정리해볼게요:

✅ 잘된 점: {요약}
🔧 개선점: {요약}
💡 인사이트: {핵심 교훈}

{피드백 기반 다음 액션 제안}"
```

## 스킵 옵션

회고를 원하지 않을 경우:

```
"괜찮아요! 나중에 하고 싶을 때 /reflect를 불러주세요.
지금은 바로 다음 단계로 넘어갈게요."
```

## 완료 조건

- 최소 1개 이상의 잘된 점 기록
- 최소 1개 이상의 개선점 기록
- `stage-{n}-reflection.md` 파일 생성
- 다음 액션 결정 (진행/반복/수정)
