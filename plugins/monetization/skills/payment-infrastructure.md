# Payment Infrastructure

Step 1.4: 결제 인프라 계획

## 설명

실제 결제를 처리할 인프라를 계획하는 스킬입니다. 타겟 시장(국내/글로벌)에 따라 PG사, 결제 플랫폼, 결제 플로우를 설계합니다.

## 실행 내용

### 질문 가이드

1. "서비스가 국내 전용인가요, 글로벌인가요?"
2. "구독 결제가 필요한가요, 일회성 결제인가요?"
3. "기술적으로 결제 연동 경험이 있나요?"

### 국내 결제 인프라

```
PG사 비교:
┌──────────────┬──────────────┬──────────────┐
│ 토스페이먼츠    │ 포트원(아임포트) │ NHN KCP      │
├──────────────┼──────────────┼──────────────┤
│ 개발자 친화적   │ 멀티 PG 통합   │ 안정적/대규모  │
│ 빠른 연동      │ 편리한 API     │ 풍부한 레퍼런스 │
│ 수수료 3.2-3.5%│ PG사 선택 가능  │ 수수료 3.2-3.5%│
│ 정기결제 지원   │ 정기결제 지원   │ 정기결제 지원  │
└──────────────┴──────────────┴──────────────┘

간편결제 연동: 카카오페이, 네이버페이, 토스, 삼성페이
부가세: 10% (가격에 포함 또는 별도 표시)
```

### 글로벌 결제 인프라

```
플랫폼 비교:
┌──────────────┬──────────────┬──────────────┐
│ Stripe        │ Paddle        │ LemonSqueezy │
├──────────────┼──────────────┼──────────────┤
│ 글로벌 표준     │ MoR (세금 대행) │ MoR (간편)    │
│ 수수료 2.9%+30¢│ 수수료 5-8%    │ 수수료 5-8%   │
│ 최대 유연성     │ VAT 자동 처리  │ 디지털 상품 최적│
│ 세금 직접 처리  │ 1인 창업자 추천 │ 소규모 추천    │
│ 130+ 통화 지원  │ 구독 관리 내장  │ 빠른 설정      │
└──────────────┴──────────────┴──────────────┘

MoR(Merchant of Record) = 세금/VAT/규정을 대신 처리
→ 1인 창업자: Paddle 또는 LemonSqueezy 강력 추천
→ 유연성 필요: Stripe + Stripe Tax
```

### 결제 플로우 설계

```
1. 체크아웃 UX:
   - 가격표 페이지 → 플랜 선택 → 결제 정보 → 완료
   - 인앱 업그레이드 → 확인 팝업 → 결제 → 기능 활성화

2. 구독 관리:
   - 플랜 변경 (업그레이드/다운그레이드)
   - 결제 수단 변경
   - 구독 일시정지 / 취소
   - 인보이스/영수증 자동 발행

3. 환불 정책:
   - 7일/14일/30일 환불 기간
   - 프로래이팅 (사용 기간 비례 환불)
```
