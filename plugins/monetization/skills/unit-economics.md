## 실행 내용

### 핵심 지표 설계

```
1. ARPU (유저당 평균 매출):
   - 가격표 기반 가중평균 산출
   - 예: Free 60% × ₩0 + Starter 30% × ₩9,900 + Pro 10% × ₩29,900

2. LTV (고객 생애 가치):
   - LTV = ARPU × 평균 고객 수명 (월)
   - 고객 수명 = 1 / 월간 이탈률
   - 예: ₩9,900 × 20개월 = ₩198,000

3. CAC (고객 획득 비용):
   - 채널별 예상 CAC 산정
   - 유기적: ₩0-5,000 / 유료 광고: ₩10,000-50,000
   - LTV/CAC 비율 목표: 3:1 이상

4. 이탈률 목표:
   - 월간 이탈률: 3.5% 이하 (SaaS 평균)
   - 목표: 2% 미만
   - 이탈 방지 전략 수립

5. 매출총이익률:
   - 서버/인프라 비용 산정
   - AI 서비스: API 호출 비용 핵심
   - 목표: 70% 이상 (성숙 SaaS 기준)

6. 페이백 기간:
   - CAC 회수까지 걸리는 기간
   - 목표: 12개월 이내
```

### MRR/ARR 시나리오

```
3가지 시나리오 (보수적 / 기본 / 낙관적):
- 3개월 목표 MRR
- 6개월 목표 MRR
- 12개월 목표 ARR
- 필요 고객 수 역산
```
