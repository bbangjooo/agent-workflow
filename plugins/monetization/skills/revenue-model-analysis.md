# Revenue Model Analysis

Step 1.1: 수익 모델 분석

## 설명

제품 특성에 적합한 수익 모델을 분석하고 최적 모델을 선택하는 스킬입니다.

## 트리거

- Stage 1 워크플로우 시작 시 첫 번째로 실행
- `/monetize` 커맨드 실행 시

## 입력

- `outputs/stage-0/idea-brief.md` — 제품 아이디어, 타겟 유저, MVP 정의
- 사용자 답변 (기존 수익 모델 아이디어, 경쟁사 과금 방식)

## 실행 내용

### 질문 가이드

1. "이 제품으로 돈을 벌 방법에 대해 생각해두신 게 있나요?"
2. "비슷한 서비스들은 보통 어떻게 과금하나요?"
3. "유저가 돈을 낼 의향이 있을까요? 무료 사용자가 더 많을까요?"
4. "B2B(기업 대상)인가요, B2C(개인 대상)인가요?"

### 분석 프레임워크

```
1. 제품 유형 분류:
   - SaaS 도구 → 구독 모델 우선 검토
   - 소비자 앱 → 프리미엄/인앱 구매
   - 마켓플레이스 → 거래 수수료
   - AI 서비스 → 크레딧/사용량 기반
   - 콘텐츠 → 광고 + 구독

2. 각 후보 모델 평가:
   - 매출 예측 가능성 (구독 > 일회성)
   - 구현 복잡도 (일회성 < 구독 < 사용량 기반)
   - 사용자 저항감 (프리미엄 < 유료 전용)
   - 스케일 가능성

3. 하이브리드 모델 검토:
   - 2025년 SaaS의 61%가 하이브리드 모델 채택
   - 예: 구독 + 사용량 초과분 과금
   - 예: 프리미엄 + 프로 구독
```
