### 1.1 Monetization Zoo (OpenView Partners)

| 항목 | 내용 |
|------|------|
| **Author/Source** | OpenView Partners (Kyle Poyar) |
| **URL** | https://openviewpartners.com/saas-pricing/ |
| **Category** | Framework |
| **Relevance** | Essential |

**핵심 내용:**
- SaaS 가격 모델을 체계적으로 분류하는 프레임워크 (subscription, usage-based, hybrid, freemium 등)
- 2025년 기준 46%의 SaaS 기업이 hybrid 모델(구독 + 사용량 기반) 채택
- $100M+ 가치 SaaS 기업의 86%가 3개 이상의 가격 차원 사용
- Multi-dimensional 모델 사용 기업은 단순 모델 대비 LTV/CAC 비율 34% 높음
- 가격 정기 검토 기업은 비검토 기업 대비 30% 높은 성장률

**Solo Founder 실전 적용:**
- 단일 가격 모델에 집착하지 말 것. 초기에는 단순 subscription으로 시작하되, PMF 이후 usage 요소 추가 고려
- 가격 모델은 최소 분기마다 재검토

### 1.2 Price Intelligently / ProfitWell Methodology

| 항목 | 내용 |
|------|------|
| **Author/Source** | Patrick Campbell (ProfitWell / Price Intelligently) |
| **URL** | https://saasclub.io/podcast/saas-pricing-patrick-campbell-price-intelligently/ |
| **Category** | Framework / Methodology |
| **Relevance** | Essential |

**4단계 SaaS 가격 프레임워크:**

1. **Buyer Persona 정의** - 고객 세그먼트별 명확한 페르소나 설정
2. **WTP(Willingness to Pay) 데이터 수집** - 4가지 핵심 설문:
   - "이 가격이면 너무 비싸서 고려하지 않을 가격은?"
   - "비싸긴 하지만 고려할 만한 가격은?"
   - "좋은 딜이라고 느끼는 가격은?"
   - "너무 싸서 품질을 의심할 가격은?"
3. **상대적 Feature Preference 분석** - 기능별 가치 측정
4. **가격 tier 정렬** - 각 페르소나가 실제로 가치 있다고 여기는 것에 가격 정렬

**핵심 원칙:**
- 가격을 수년간 그대로 두면 막대한 수익 손실 (제품 개선, 브랜드 성장, 시장 변화 반영 안됨)
- Value-based pricing이 핵심: 구매자 페르소나를 정량화하여 수익 누수 방지
- 분기마다 가격 재검토 권장

**Solo Founder 실전 적용:**
- 초기 10-20명의 잠재 고객에게 4가지 WTP 설문 실시
- 그래프로 그리면 가격 탄력성과 최적 가격 범위가 나옴

### 1.3 Madhavan Ramanujam's "Monetizing Innovation" (WTP Framework)

| 항목 | 내용 |
|------|------|
| **Author/Source** | Madhavan Ramanujam & Georg Tacke (Simon-Kucher & Partners) |
| **URL** | https://www.amazon.com/Monetizing-Innovation-Companies-Design-Product/dp/1119240867 |
| **Category** | Framework / Book |
| **Relevance** | Essential |

**핵심 문제 인식:**
- 혁신의 72%가 재무 목표 미달 또는 완전 실패
- 대부분 기업이 제품을 먼저 만들고, 가격을 나중에 억지로 맞추기 때문

**WTP Framework 핵심:**

1. **제품 개발 초기에 WTP 대화** - 가격이 제품 설계를 주도해야 함 ("Price before product")
2. **WTP 기반 세그멘테이션** - 인구통계가 아닌 지불 의향 기준으로 세분화
3. **기능별 WTP 파악** - 각 기능에 대한 지불 의향을 개별적으로 측정하여 로드맵 우선순위, feature shock 방지, 세그멘테이션 결정
4. **올바른 Pricing Model 선택** - "얼마를 받느냐"보다 "어떻게 받느냐"가 더 중요
5. **WTP 데이터 기반 비즈니스 케이스** - 고객 지불 의향 데이터에 기반한 사업 계획 수립

**실전 사례: Porsche Cayenne**
- 제품 설계 전에 타겟 고객에게 가능한 모든 기능에 대한 WTP 조사
- 고객이 가치를 인정하고 지불 의향이 있는 기능만 포함
- 결과: Porsche SUV에 대해 프리미엄 가격을 지불할 것이라는 확신 확보

**Solo Founder 실전 적용:**
- MVP 전에 "고객이 이것에 돈을 낼까?"를 먼저 검증
- 기능 리스트를 만들고 각 기능별 WTP를 조사하여 MVP 범위 결정

### 1.6 Value Metric Framework

| 항목 | 내용 |
|------|------|
| **Author/Source** | Patrick Campbell (ProfitWell) / OpenView Partners |
| **Category** | Framework |
| **Relevance** | Essential |

**핵심 개념:**
- Value Metric = "무엇에 대해 과금할 것인가"의 단위
- 좋은 Value Metric의 3가지 조건:
  1. 고객이 받는 가치와 정렬
  2. 사용량이 증가할수록 과금도 증가 (자연스러운 확장)
  3. 이해하기 쉬움

**예시:**
- Slack: 활성 사용자 수 (active users)
- AWS: 컴퓨팅 시간, 스토리지 용량
- Stripe: 처리된 결제 건수
- Mailchimp: 구독자 수
- Figma: 에디터 수

**Solo Founder 실전 적용:**
- "고객이 우리 제품에서 가장 가치를 느끼는 순간은 언제인가?"를 기준으로 과금 단위 결정
- Per-seat이 항상 답이 아님. AI 시대에는 특히 outcome/usage 기반 고려

### 1.7 Jobs to be Done (JTBD) for Pricing

| 항목 | 내용 |
|------|------|
| **Author/Source** | Clayton Christensen, Tony Ulwick, Bob Moesta |
| **URL** | https://www.christenseninstitute.org/theory/jobs-to-be-done/ |
| **Category** | Framework |
| **Relevance** | Recommended |

**가격 결정에 JTBD 적용:**
- 가격 메트릭을 고객이 수행하는 "job"의 가치 단위에 정렬
- 예: 위치당(per location), 달성된 결과당(per outcome)
- 기능이나 제품 속성이 아닌, 고객이 실제로 달성하는 결과에 기반한 가격 구조
- Outcome-based positioning: job을 명명하고, outcome을 정량화하고, 실제 대안과 비교

**핵심 질문:** "사람들이 더 나은 job 완수를 위해 더 많이 지불할 의향이 있는 곳은 어디인가?"

**Solo Founder 실전 적용:**
- "고객이 우리 제품을 '고용'하는 이유"를 파악하면 가격 정당성이 자연스럽게 나옴
- 보장, 벤치마크, 검증된 결과를 통해 가격에 대한 신뢰를 구축

### 1.8 Pirate Metrics (AARRR) - Dave McClure

| 항목 | 내용 |
|------|------|
| **Author/Source** | Dave McClure (500 Startups) |
| **URL** | https://amplitude.com/blog/pirate-metrics-framework |
| **Category** | Framework |
| **Relevance** | Recommended |

**5가지 지표:**
1. **Acquisition** - 사람들이 어떻게 우리 제품을 발견하는가?
2. **Activation** - 원하는 행동을 취하고 있는가?
3. **Retention** - 활성화된 사용자가 계속 사용하는가?
4. **Referral** - 다른 사람에게 추천할 만큼 좋아하는가?
5. **Revenue** - 이 제품에 돈을 지불할 의향이 있는가?

**McClure의 핵심 철학:**
- 제품 개발 초기부터 확장성과 수익화를 우선시해야 함
- 고객의 문제를 해결하고, 그 해결책을 수익화하고, 제공 비용보다 더 많은 수익을 창출해야 함
- 소셜 미디어 좋아요 같은 표면적 지표에 현혹되지 말 것

**Solo Founder 실전 적용:**
- Revenue는 마지막 단계이지만, 초기부터 설계에 포함되어야 함
- Activation과 Retention이 먼저 안정되어야 Revenue 최적화가 의미 있음

### 1.9 The Mom Test - Rob Fitzpatrick

| 항목 | 내용 |
|------|------|
| **Author/Source** | Rob Fitzpatrick |
| **URL** | https://www.momtestbook.com |
| **Category** | Book / Methodology |
| **Relevance** | Essential |

**WTP 검증 핵심 원칙:**

1. **아이디어가 아닌 고객의 삶에 대해 질문** - 아이디어 이야기를 멈추고 그들의 삶에 대해 질문 시작
2. **Commitment으로 진지함 측정** - 시간, 평판, 돈 등 가치 있는 것을 포기하는 것이 진정한 관심의 증거
   - 포기하는 것이 클수록 말의 진정성이 높음
3. **과거 행동에 집중** - 가상의 미래 의도가 아닌 과거 행동을 물어야 신뢰성 있는 구매/가격 신호 획득
4. **"Nice-to-have" 함정 회피** - "불편하지만 결국 해결됨" = 시장 수요가 얕음

**Solo Founder 실전 적용:**
- "이거 쓸래요?"보다 "지금 이 문제를 어떻게 해결하고 있어요?"
- "돈 낼 의향이 있어요?"보다 "이 문제 해결에 지금까지 얼마나 썼어요?"
- Pre-order, Letter of Intent, 유료 파일럿 등 실제 commitment 요구

### 2.4 AI SaaS Pricing Models (2026)

| 항목 | 내용 |
|------|------|
| **Sources** | Bessemer VP, Chargebee, Deloitte, Monetizely |
| **URLs** | https://www.bvp.com/atlas/the-ai-pricing-and-monetization-playbook / https://www.getmonetizely.com/blogs/the-2026-guide-to-saas-ai-and-agentic-pricing-models |
| **Category** | Research / Trend |
| **Relevance** | Essential |

**2026 AI 가격 모델 트렌드:**

- **Per-seat의 종말**: Agentic AI가 인간 로그인 수요를 감소시키면서 per-seat 모델 위협
- **Outcome-Based Pricing 부상**: 현재 10% 미만 채택, but 2025년까지 ~30% 도달 예상
  - 예: Intercom - AI가 고객 대화를 완전 해결할 때만 $0.99 과금 (실패 시 무료)
  - 첫 해에 수천만 달러 수익 창출
- **Credit System 확산**:
  - Workday: "Flex Credits" - AI 결과물/에이전트별 크레딧 소비
  - Adobe: "Generative Credit" - 특정 output에 대한 크레딧 과금
- **Hybrid가 지배적**: Base subscription + usage allowance, 또는 per-seat + fair use limit

**Solo Founder 실전 적용:**
- AI 기반 제품이라면 token/credit 기반 usage pricing 고려
- 고정 base + 사용량 초과분 과금 = 예측 가능성 + 성장 정렬
- Outcome-based pricing은 AI 성능에 자신 있을 때 강력한 차별화

## 3. Pricing Psychology & Behavioral Economics

### 3.1 Dan Ariely's "Predictably Irrational" - Decoy Effect

| 항목 | 내용 |
|------|------|
| **Author** | Dan Ariely |
| **Category** | Book / Research |
| **Relevance** | Essential |

**The Economist 실험 (가장 유명한 가격 심리 사례):**

| 옵션 | 가격 | Decoy 없을 때 선택률 | Decoy 있을 때 선택률 |
|------|------|---------------------|---------------------|
| Web only | $59 | 68% | 16% |
| Print only (Decoy) | $125 | - | 0% |
| Web + Print | $125 | 32% | **84%** |

- Decoy가 없으면 32%만 비싼 옵션 선택
- Decoy 추가 시 84%가 비싼 옵션 선택 (163% 증가)

**Solo Founder 실전 적용:**
- 3-tier 가격표에서 중간 tier가 타겟이라면, 상위 tier를 decoy로 설계
- "Print only = $125" 같은 명백히 열등한 옵션이 비교 기준점을 바꿈

### 3.4 Endowment Effect & Free Trials

| 항목 | 내용 |
|------|------|
| **Category** | Behavioral Economics |
| **Relevance** | Recommended |

**핵심 원리:**

## Quick Reference: Solo Founder를 위한 수익화 체크리스트

### Phase 1: Validation (제품 전)
- [ ] The Mom Test로 고객 인터뷰 (과거 행동, 현재 해결책에 집중)
- [ ] WTP 설문 (4가지 가격 질문) 최소 10-20명
- [ ] Lean Canvas 작성 (Problem → Solution → Revenue Streams)
- [ ] JTBD 관점: "고객이 이 제품을 고용하는 이유" 정의
- [ ] Pre-order/LOI/유료 파일럿으로 commitment 확보

### Phase 2: Initial Pricing (MVP ~ PMF)
- [ ] Value Metric 결정 (과금 단위)
- [ ] 3-tier 가격 구조 설계 (Decoy effect 활용)
- [ ] Free trial vs Freemium 결정 (바이럴 루프 가능? → Freemium)
- [ ] Sean Ellis PMF 테스트 (40% "매우 실망" 기준)
- [ ] AARRR funnel 기본 측정 설정

### Phase 3: Optimization (PMF 이후)
- [ ] 분기별 가격 재검토 (ProfitWell methodology)
- [ ] A/B 테스트로 가격 페이지 최적화
- [ ] Usage-based 또는 credit 요소 추가 검토
- [ ] Expansion revenue 최적화 (upgrade trigger 설계)
- [ ] Unit economics 추적 (CAC, LTV, Payback, NRR)

### Phase 4: Scale
- [ ] Enterprise tier 추가
- [ ] Annual discount으로 현금 흐름 개선
- [ ] Fair billing 또는 usage credit 도입으로 신뢰 구축
- [ ] Multi-dimensional pricing 도입
